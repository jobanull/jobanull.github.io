Cisco ISE (Identity Services Engine)

Cisco ISE adalah solusi keamanan jaringan yang berfungsi sebagai pusat otentikasi, otorisasi, dan akuntansi (AAA) untuk perangkat yang terhubung ke jaringan. ISE memungkinkan administrator mengontrol siapa yang boleh mengakses jaringan, perangkat apa yang digunakan, serta tingkat akses yang diberikan, sehingga meningkatkan keamanan dan kepatuhan kebijakan perusahaan.

Fungsi Utama Cisco ISE

  • Melakukan otentikasi pengguna dan perangkat sebelum mengakses jaringan.
  • Menerapkan otorisasi berbasis kebijakan (policy-based access control).
  • Menyediakan akuntansi untuk mencatat aktivitas akses jaringan.
  • Integrasi dengan profiling perangkat IoT, BYOD, dan endpoint lain.
  • Integrasi dengan solusi keamanan lain seperti Cisco TrustSec, Firepower, dan Umbrella.

Komponen Utama Cisco ISE

  1. Policy Service Node (PSN) – menangani permintaan otentikasi dan otorisasi.
  2. Administration Node – pusat manajemen dan konfigurasi kebijakan.
  3. Monitoring & Troubleshooting Node (MnT) – menyimpan log dan laporan aktivitas.
  4. Profiling Service – mengidentifikasi dan mengklasifikasikan perangkat di jaringan.

Kelebihan Cisco ISE

  • Meningkatkan keamanan jaringan dengan kontrol akses berbasis identitas.
  • Mendukung 802.1X untuk autentikasi jaringan tingkat lanjut.
  • Dapat diintegrasikan dengan Active Directory dan LDAP.
  • Memberikan visibilitas penuh terhadap perangkat dan pengguna di jaringan.
  • Memperkuat kepatuhan terhadap standar keamanan (compliance).

Kekurangan Cisco ISE

  • Kompleksitas implementasi cukup tinggi.
  • Memerlukan lisensi tambahan untuk fitur tertentu (misalnya profiling, posture).
  • Biaya investasi relatif mahal.

Alasan Kenapa Harus Pakai Cisco ISE

  • Keamanan lebih tinggi – hanya user dan perangkat sah yang bisa masuk jaringan.
  • Kontrol akses fleksibel – bisa dibatasi berdasarkan identitas, role, atau perangkat.
  • Visibilitas penuh – admin tahu siapa dan apa yang terhubung, termasuk status perangkat.
  • Integrasi luas – dapat dihubungkan dengan AD/LDAP, firewall, NAC, dan perangkat Cisco lainnya.
  • Efisiensi operasional – kebijakan dikelola terpusat, mempermudah manajemen skala besar.

Contoh Kasus

Tanpa Cisco ISE – semua orang yang tahu password Wi-Fi bisa langsung masuk jaringan, tamu bisa akses resource internal, admin sulit memantau siapa saja yang terhubung, dan rawan serangan dari perangkat tidak aman.

Dengan Cisco ISE – setiap user harus login sesuai identitas (misalnya akun kampus/perusahaan), tamu diarahkan ke portal guest Wi-Fi, perangkat dicek antivirus & update sebelum diizinkan, admin bisa melihat dashboard lengkap serta mengisolasi perangkat mencurigakan hanya dengan klik.

  • Kampus: Mahasiswa hanya akses e-learning, dosen ke server akademik, tamu hanya internet.
  • Perusahaan: Staff HRD hanya akses aplikasi HR, IT punya akses penuh, tamu diarahkan ke internet saja.

Contoh Konfigurasi Radius pada Cisco ISE


# Konfigurasi pada Cisco Switch untuk integrasi dengan ISE

radius server ISE
 address ipv4 192.168.10.10 auth-port 1812 acct-port 1813
 key cisco123

ip radius source-interface GigabitEthernet0/1

# Aktifkan AAA
aaa new-model
aaa authentication dot1x default group radius
aaa authorization network default group radius
aaa accounting update periodic 5
aaa accounting dot1x default start-stop group radius

# Konfigurasi 802.1X di interface
interface GigabitEthernet0/2
 switchport mode access
 authentication port-control auto
 mab
 dot1x pae authenticator

Tips Praktis

  • Gunakan integrasi dengan Active Directory untuk mempermudah manajemen user.
  • Aktifkan profiling agar perangkat non-user (printer, IP phone, IoT) tetap terkelola.
  • Terapkan segmentasi dengan Cisco TrustSec untuk keamanan lebih granular.
  • Lakukan monitoring pada MnT node untuk audit keamanan jaringan.

Quality of Service (Simple Queue)

Simple Queue adalah metode QoS dasar di MikroTik untuk mengatur bandwidth secara langsung per user, subnet, atau interface. Cocok untuk jaringan kecil hingga menengah yang ingin membatasi atau membagi bandwidth dengan cepat dan mudah.

  • Kelebihan: Mudah di-setup, langsung mengatur limit bandwidth per target; cocok untuk jaringan kecil atau kantor dengan sedikit user/subnet.
  • Keterbatasan: Kurang fleksibel untuk jaringan kompleks; tidak sebaik PCQ atau Q-Tree untuk distribusi per koneksi/user banyak.
  • Gunakan Saat: Membatasi bandwidth user tertentu, subnet kantor, atau interface WAN agar tidak melebihi kapasitas link.

CONTOH KASUS

  • Membatasi bandwidth client dengan Simple Queue.
  • Subnet 192.168.1.0/30 dibatasi total 1 Mbps (parent queue).
  • Client 192.168.1.2 hanya dapat 254 kbps (child queue).
  • Client 192.168.2.2 punya limit sendiri 1 Mbps.
  • Semua client keluar internet melalui NAT di ether1.

R.SQ


/queue simple
add max-limit=1M/1M name=queue-parent target=192.168.1.0/30           # Parent queue untuk subnet 192.168.1.0/30 dengan total limit 1 Mbps up/down
add max-limit=254k/254k name=child-queue1 parent=queue-parent target=\
    192.168.1.2/32                                                     # Queue anak untuk client dengan IP 192.168.1.2, limit 254 kbps up/down
add max-limit=1M/1M name=queue-ether3 target=192.168.2.2/32           # Queue untuk client dengan IP 192.168.2.2, limit 1 Mbps up/down

/ip address
add address=192.168.1.1/30 interface=ether2 network=192.168.1.0       # IP gateway client1 (subnet 192.168.1.0/30) di ether2
add address=192.168.2.1/30 interface=ether3 network=192.168.2.0       # IP gateway client2 (subnet 192.168.2.0/30) di ether3

/ip dhcp-client
add disabled=no interface=ether1                                      # Interface WAN (ether1) mendapat IP otomatis dari ISP via DHCP

/ip firewall nat
add action=masquerade chain=srcnat out-interface=ether1               # NAT agar client bisa akses internet keluar via ether1

/system identity
set name=R.SQ                                                         # Nama router = R.SQ

Tips Praktis

  • Gunakan Simple Queue untuk jaringan kecil agar mudah dikelola.
  • Untuk banyak user, pertimbangkan PCQ agar distribusi bandwidth lebih adil otomatis.
  • Monitor antrean dengan `/queue simple print` untuk memastikan limit bekerja sesuai rencana.
  • Dapat dipadukan dengan burst agar user mendapat bandwidth lebih tinggi sesaat saat trafik rendah.

Ringkasan

Simple Queue adalah cara cepat dan mudah untuk membatasi atau membagi bandwidth per user, subnet, atau interface. Ideal untuk jaringan kecil sampai menengah yang membutuhkan kontrol bandwidth langsung tanpa konfigurasi kompleks.

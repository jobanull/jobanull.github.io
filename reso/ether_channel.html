EtherChannel (Link Aggregation)

EtherChannel adalah teknologi yang digunakan untuk menggabungkan beberapa link fisik menjadi satu link logis dengan tujuan meningkatkan bandwidth, menyediakan redundansi, dan load balancing. EtherChannel banyak digunakan pada switch Cisco dan perangkat jaringan lainnya untuk efisiensi dan ketahanan koneksi.

Fungsi Utama EtherChannel

  • Peningkatan Bandwidth - Menggabungkan beberapa link menjadi satu jalur logis.
  • Load Balancing - Distribusi lalu lintas antar-link fisik.
  • Redundansi - Jika salah satu link gagal, trafik otomatis dialihkan ke link lain.
  • Manajemen Lebih Mudah - Beberapa link dikonfigurasi sebagai satu interface logis (Port-Channel).

Jenis EtherChannel

  • Static (On) - Konfigurasi manual tanpa protokol negosiasi.
  • PAgP (Port Aggregation Protocol) - Protokol proprietary Cisco.
  • LACP (Link Aggregation Control Protocol) - Standar IEEE 802.3ad, interoperable dengan vendor lain.

Kelebihan EtherChannel

  • Meningkatkan throughput dengan menggabungkan beberapa link.
  • Menyediakan fault-tolerance dan failover otomatis.
  • Mengurangi kebutuhan spanning-tree memblokir port, karena link digabung.
  • Menyederhanakan administrasi jaringan (satu port-channel menggantikan banyak interface).

Kekurangan EtherChannel

  • Memerlukan perangkat dengan dukungan EtherChannel/LACP.
  • Link fisik harus memiliki kecepatan dan konfigurasi yang sama.
  • Salah konfigurasi dapat menyebabkan loop jaringan.

CONTOH KASUS

  • Tujuan: Mengonfigurasi EtherChannel pada Cisco Switch untuk meningkatkan bandwidth antar switch core dan distribusi.
  • Cisco: Menggabungkan beberapa port FastEthernet/GigabitEthernet menjadi satu port-channel dengan LACP atau PAgP.

Cisco (LACP)


interface range gigabitEthernet 0/1 - 2
 channel-group 1 mode active   ! mode active = LACP
 switchport mode trunk
!
interface port-channel 1
 switchport mode trunk

Cisco (PAgP)


interface range fastEthernet 0/3 - 4
 channel-group 2 mode desirable   ! mode desirable = PAgP
 switchport mode access
!
interface port-channel 2
 switchport mode access
 switchport access vlan 20

Cisco (Static)


interface range gigabitEthernet 0/5 - 6
 channel-group 3 mode on   ! tanpa protokol, langsung static
 switchport mode trunk
!
interface port-channel 3
 switchport mode trunk

Tips Praktis

  • Pastikan konfigurasi di kedua sisi EtherChannel sama (speed, duplex, VLAN, mode).
  • Gunakan LACP untuk interoperabilitas antar vendor.
  • Gunakan perintah show etherchannel summary untuk troubleshooting.
  • Hindari mencampur port dengan jenis/kecepatan berbeda dalam satu channel-group.

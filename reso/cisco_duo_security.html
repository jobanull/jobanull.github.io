Cisco Duo Security

Cisco Duo Security adalah platform keamanan yang berfokus pada **Zero Trust** dan autentikasi multifaktor (MFA). Tujuannya adalah untuk memverifikasi identitas pengguna dan kesehatan perangkat mereka sebelum memberikan akses ke aplikasi dan data perusahaan. Duo Security melindungi dari serangan phishing dan kredensial yang dicuri dengan memastikan bahwa hanya pengguna yang terautentikasi dan perangkat yang tepercaya yang dapat terhubung ke sumber daya jaringan.

Fungsi Utama Cisco Duo Security

  • Autentikasi Multifaktor (MFA) – Membutuhkan lebih dari satu metode verifikasi untuk membuktikan identitas pengguna, seperti password dan push notifikasi ke ponsel.
  • Verifikasi Perangkat (Device Trust) – Memeriksa kesehatan perangkat yang mengakses jaringan (misalnya, apakah perangkat sudah diperbarui, memiliki enkripsi, atau tidak memiliki malware).
  • Akses Tanpa Agen (Agentless Access) – Memberikan perlindungan ke aplikasi cloud dan on-premises tanpa perlu menginstal software khusus di setiap perangkat.
  • Visibilitas dan Analitik – Menyediakan visibilitas terperinci tentang perangkat dan aktivitas pengguna, membantu mengidentifikasi potensi ancaman.

Jenis Metode Autentikasi Duo

  • Duo Push – Notifikasi yang dikirim ke aplikasi Duo di ponsel untuk verifikasi satu ketukan. Ini adalah metode yang paling mudah dan disarankan.
  • One-Time Password (OTP) – Kode sandi yang dibuat oleh aplikasi atau token hardware Duo.
  • Telepon dan SMS – Panggilan telepon atau kode SMS yang dikirim ke nomor yang terdaftar.
  • Biometrik – Menggunakan sidik jari atau pengenalan wajah yang sudah ada di perangkat.

Kelebihan Cisco Duo Security

  • Sangat mudah digunakan, meningkatkan adopsi MFA di kalangan pengguna.
  • Mengurangi risiko kredensial yang dicuri secara signifikan.
  • Fleksibel dan dapat diintegrasikan dengan berbagai aplikasi dan platform, baik cloud maupun on-premises.
  • Menyediakan visibilitas yang kuat ke dalam postur keamanan perangkat.

Kekurangan Cisco Duo Security

  • Harga dapat menjadi mahal seiring dengan bertambahnya jumlah pengguna.
  • Pengguna memerlukan perangkat seluler untuk menggunakan metode MFA yang paling aman (Duo Push).
  • Beberapa metode autentikasi (seperti SMS) masih rentan terhadap serangan SIM swapping.
  • Membutuhkan koneksi internet untuk metode MFA berbasis cloud (Duo Push).

CONTOH KASUS

  • Tujuan: Mengamankan akses remote ke server perusahaan menggunakan **Cisco Duo Security**.
  • Cisco Duo: Setelah pengguna memasukkan username dan password mereka, mereka akan menerima notifikasi "Push" di ponsel mereka. Akses hanya diberikan setelah mereka menyetujui notifikasi tersebut.

Contoh Konsep Integrasi (Konfigurasi di Server VPN)


// Catatan: Konfigurasi Duo dilakukan melalui GUI.
// Contoh berikut adalah konsep integrasi RADIUS untuk VPN.

// 1. Konfigurasi Server Duo Authentication Proxy
radius_client:
  host=203.0.113.1
  radius_secret=mysupersecretkey

radius_server_auto:
  ikey=DIXXXXXXXXXXXXXXXXXX
  skey=DIXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
  api_host=api-hostname.duosecurity.com

// 2. Konfigurasi Client VPN (misal: Cisco ASA)
aaa-server DUO protocol radius
 aaa-server DUO host 10.0.1.10
  key mysupersecretkey

tunnel-group MYVPN webvpn-attributes
  authentication-server-group DUO

Tips Praktis

  • Dorong pengguna untuk menggunakan metode **Duo Push** karena paling aman dan mudah.
  • Manfaatkan fitur **Device Trust** untuk memastikan perangkat yang mengakses jaringan memenuhi standar keamanan minimum.
  • Lakukan pelatihan rutin kepada pengguna tentang cara mengenali dan menghindari serangan phishing.
  • Gunakan Duo untuk melindungi semua akses sensitif, termasuk server, VPN, dan aplikasi cloud.

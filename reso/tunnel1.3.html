IPIP adalah tunnel layer 3 sederhana yang mengenkapsulasi paket IP di dalam paket IP lain. Cocok untuk menghubungkan subnet antar lokasi berbeda melalui jaringan IP tanpa enkripsi.

  • Kelebihan: Ringan, mudah dikonfigurasi, overhead rendah, mendukung routing antar subnet.
  • Keterbatasan: Tidak terenkripsi, hanya transport IP (tidak mengangkut layer 2 atau VLAN).
  • Gunakan Saat: Site-to-site routing antar jaringan IP, menghubungkan subnet berbeda secara langsung, transport IP sederhana antar MikroTik atau perangkat lain yang mendukung IPIP.

    192.168.190.103

set [ find default=yes ] supplicant-identity=MikroTik
/ip address
add address=192.168.190.102/24 interface=ether1 network=192.168.190.0
add address=192.168.1.1/30 interface=ether2 network=192.168.1.0
add address=10.10.10.1/30 interface=ipip-tunnel1 network=10.10.10.0
/ip dns
set servers=8.8.8.8,8.8.4.4
/ip firewall nat
add action=masquerade chain=srcnat out-interface=ether1
/ip route
add distance=1 gateway=192.168.190.1
add distance=1 dst-address=192.168.2.0/30 gateway=10.10.10.2

Tips Praktis IPIP

  • IPIP hanya mengangkut paket IP; jika butuh bridging layer 2, gunakan EOIP atau VXLAN.
  • Pastikan IP lokal/remote tunnel dapat saling jangkau melalui jaringan fisik.
  • Gunakan route statik untuk subnet yang ingin dilewatkan melalui tunnel IPIP.
  • Untuk keamanan di jaringan publik, kombinasikan IPIP dengan IPsec.
  • Overhead IPIP = 20 bytes per paket, perhatikan MTU jika melewati jaringan internet.

Ringkasan

IPIP adalah tunnel layer 3 non-VPN yang sederhana dan ringan untuk site-to-site routing antar subnet IP. Tidak terenkripsi, tapi mudah dikonfigurasi dan kompatibel dengan perangkat lain yang mendukung IPIP.

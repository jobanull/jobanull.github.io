Embedded Event Manager (EEM)

EEM (Embedded Event Manager) adalah fitur bawaan Cisco IOS/IOS-XE/IOS-XR yang memungkinkan perangkat jaringan (router, switch, firewall) untuk melakukan monitoring, otomatisasi, dan skripting berdasarkan event tertentu tanpa harus menggunakan server manajemen eksternal.

Fungsi Utama EEM

  • Monitoring ‚Äì Mengawasi kondisi tertentu, misalnya interface down, CPU tinggi, syslog, atau SNMP trap.
  • Automation ‚Äì Menjalankan aksi otomatis seperti mengeksekusi perintah CLI, reset interface, atau kirim notifikasi.
  • Scripting ‚Äì Mendukung skrip sederhana dengan applet berbasis CLI atau skrip kompleks dengan TCL.

Komponen EEM

  • Event Detector ‚Äì Pemicu event (timer, syslog, CLI, interface, SNMP).
  • Action ‚Äì Tindakan setelah event terdeteksi (jalankan perintah, kirim log, aktifkan ulang interface).
  • Policy ‚Äì Aturan yang menentukan hubungan event dan action (Applet atau TCL script).

Contoh Konfigurasi

1. Monitoring interface down lalu kirim syslog


event manager applet IF_DOWN
 event syslog pattern "Interface GigabitEthernet0/0, changed state to administratively down"
 action 1.0 syslog msg "Warning: Gi0/0 down!"

2. Auto-restart interface jika link mati


event manager applet RESTART_IF
 event syslog pattern "Interface GigabitEthernet0/1, changed state to down"
 action 1.0 cli command "enable"
 action 2.0 cli command "conf t"
 action 3.0 cli command "interface Gi0/1"
 action 4.0 cli command "shutdown"
 action 5.0 wait 10
 action 6.0 cli command "no shutdown"

3. Menjalankan perintah rutin tiap 5 menit


event manager applet SHOW_IP
 event timer watchdog time 300
 action 1.0 cli command "show ip interface brief"

Kegunaan di Dunia Nyata

  • Auto-recovery ‚Äì Memulihkan link secara otomatis saat down.
  • Proactive monitoring ‚Äì Memberikan alert/log tanpa perlu NMS eksternal.
  • Scheduled task ‚Äì Menjalankan perintah berkala (mirip cron di Linux).
  • Integrasi ‚Äì Bisa bekerja sama dengan syslog, SNMP, atau policy keamanan.

Tips Praktis

  • Gunakan EEM untuk otomatisasi ringan, bukan pengganti penuh NMS.
  • Hati-hati dengan aksi otomatis seperti reload/shutdown, pastikan ada delay.
  • Dokumentasikan semua applet EEM agar mudah di-maintain oleh tim lain.

Lab 1 ‚Äì VLAN dan Trunk üñß

üìå Penjelasan Singkat

VLAN (Virtual LAN) adalah cara memisahkan jaringan secara logis di Layer 2, walau perangkat tetap terhubung ke switch yang sama.

Tujuan: segmentasi jaringan, keamanan, efisiensi broadcast.

Trunking memungkinkan satu link switch membawa traffic beberapa VLAN sekaligus. Standar yang umum: IEEE 802.1Q. Setiap frame diberi tag VLAN supaya switch tujuan bisa mengenali VLAN-nya.

üñ•Ô∏è Contoh Kasus

Sebuah kantor memiliki 3 departemen: HR, Finance, IT. Tiap departemen perlu VLAN berbeda:

  • VLAN 10 ‚Üí HR
  • VLAN 20 ‚Üí Finance
  • VLAN 30 ‚Üí IT

Traffic antar switch tetap harus bisa membawa semua VLAN ‚Üí perlu trunk link.

‚öôÔ∏è Konfigurasi

Step 1 ‚Äì Buat VLAN

Switch(config)# vlan 10
Switch(config-vlan)# name HR
Switch(config-vlan)# exit

Switch(config)# vlan 20
Switch(config-vlan)# name Finance
Switch(config-vlan)# exit

Switch(config)# vlan 30
Switch(config-vlan)# name IT
Switch(config-vlan)# exit
Step 2 ‚Äì Assign VLAN ke port

Switch(config)# interface range fa0/1-5
Switch(config-if-range)# switchport mode access
Switch(config-if-range)# switchport access vlan 10

Switch(config)# interface range fa0/6-10
Switch(config-if-range)# switchport mode access
Switch(config-if-range)# switchport access vlan 20

Switch(config)# interface range fa0/11-15
Switch(config-if-range)# switchport mode access
Switch(config-if-range)# switchport access vlan 30
Step 3 ‚Äì Konfigurasi Trunk (antar switch)

Switch(config)# interface gi0/1
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk allowed vlan 10,20,30
Step 4 ‚Äì Verifikasi
  • Cek VLAN: show vlan brief
  • Cek trunk: show interfaces trunk

‚úÖ Hasil yang Diharapkan

  • Port sudah tergabung sesuai VLAN masing-masing departemen
  • Trunk antar switch bisa membawa traffic semua VLAN
  • Segmentasi jaringan berjalan sesuai rencana

Lab 2 ‚Äì InterVLAN Routing üîÄ

üìå Penjelasan Singkat

Masalah: VLAN memisahkan jaringan di Layer 2, sehingga perangkat di VLAN berbeda tidak bisa saling berkomunikasi secara langsung.

Solusi: InterVLAN Routing menggunakan perangkat Layer 3 (Router atau Layer 3 Switch) untuk menghubungkan VLAN berbeda.

Metode umum:

  • Router-on-a-stick: Router memiliki sub-interface untuk tiap VLAN, biasanya melalui trunk link.
  • Layer 3 Switch: Switch memiliki SVI (Switch Virtual Interface) untuk tiap VLAN, bisa routing langsung tanpa router tambahan.

üñ•Ô∏è Contoh Kasus

Kantor sama seperti Lab 1: HR (VLAN 10), Finance (VLAN 20), IT (VLAN 30). Tujuannya: PC di HR bisa mengakses PC di Finance dan IT, tapi tetap menjaga segmentasi VLAN.

‚öôÔ∏è Konfigurasi

1Ô∏è‚É£ Router-on-a-Stick

Router(config)# interface gi0/0.10
Router(config-subif)# encapsulation dot1Q 10
Router(config-subif)# ip address 192.168.10.1 255.255.255.0

Router(config)# interface gi0/0.20
Router(config-subif)# encapsulation dot1Q 20
Router(config-subif)# ip address 192.168.20.1 255.255.255.0

Router(config)# interface gi0/0.30
Router(config-subif)# encapsulation dot1Q 30
Router(config-subif)# ip address 192.168.30.1 255.255.255.0

Pastikan trunk antara switch dan router: 


Switch(config)# interface gi0/1
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk allowed vlan 10,20,30
2Ô∏è‚É£ Layer 3 Switch (SVI)

Switch(config)# interface vlan 10
Switch(config-if)# ip address 192.168.10.1 255.255.255.0
Switch(config-if)# no shutdown

Switch(config)# interface vlan 20
Switch(config-if)# ip address 192.168.20.1 255.255.255.0
Switch(config-if)# no shutdown

Switch(config)# interface vlan 30
Switch(config-if)# ip address 192.168.30.1 255.255.255.0
Switch(config-if)# no shutdown

Switch(config)# ip routing
üîç Verifikasi

Switch# ping 192.168.20.1   # dari VLAN 10 ke VLAN 20
Switch# ping 192.168.30.1   # dari VLAN 10 ke VLAN 30

‚úÖ Hasil yang Diharapkan

  • Perangkat di VLAN berbeda bisa saling berkomunikasi
  • Segmentasi VLAN tetap terjaga
  • Router-on-a-stick atau Layer 3 Switch berfungsi sebagai gateway antar VLAN

Lab 3 ‚Äì SPAN (Switch Port Analyzer) üîç

üìå Penjelasan Singkat

SPAN digunakan untuk memantau traffic di satu atau beberapa port switch tanpa mengganggu aliran data normal.

Traffic dari source port akan dicopy ke destination port, yang biasanya terhubung ke perangkat monitoring seperti Wireshark atau IDS.

SPAN bersifat lokal, artinya hanya memonitor port di switch yang sama.

üñ•Ô∏è Contoh Kasus

IT ingin memantau semua traffic yang masuk ke server VLAN 20 untuk analisis keamanan.

Server terhubung ke port Fa0/10, dan kita ingin copy traffic ke port Fa0/20 yang terhubung ke laptop Wireshark.

‚öôÔ∏è Konfigurasi


Switch(config)# monitor session 1 source interface fa0/10
Switch(config)# monitor session 1 destination interface fa0/20

üîç Verifikasi


Switch# show monitor

Sekarang semua traffic dari Fa0/10 akan dicopy ke Fa0/20 dan bisa di-capture di Wireshark.

‚úÖ Hasil yang Diharapkan

  • Port source dapat dimonitor tanpa mengganggu aliran data normal
  • Port destination menerima salinan traffic untuk analisis
  • Monitoring lokal berjalan efektif menggunakan SPAN

Lab 4 ‚Äì RSPAN (Remote SPAN) üåê

üìå Penjelasan Singkat

RSPAN adalah pengembangan SPAN yang memungkinkan monitoring traffic dari switch lain melalui jaringan.

Traffic dari source port dikirim melalui VLAN khusus RSPAN ke switch lain, lalu dicapture di destination port.

Membutuhkan RSPAN VLAN khusus yang dikonfigurasi di semua switch yang terlibat.

üñ•Ô∏è Contoh Kasus

Perusahaan memiliki 2 switch: Switch A dan Switch B. Server VLAN 20 di Switch A ingin dipantau dari Switch B, tempat analyzer Wireshark terhubung.

Solusi: buat RSPAN VLAN 999, traffic dari server di Switch A dikirim ke Switch B melalui VLAN ini.

‚öôÔ∏è Konfigurasi

1Ô∏è‚É£ Buat RSPAN VLAN

Switch(config)# vlan 999
Switch(config-vlan)# remote-span
Switch(config-vlan)# exit
2Ô∏è‚É£ Konfigurasi Source Port (Switch A)

Switch(config)# monitor session 1 source interface fa0/10
Switch(config)# monitor session 1 destination remote vlan 999
3Ô∏è‚É£ Konfigurasi Destination Port (Switch B)

Switch(config)# monitor session 1 source remote vlan 999
Switch(config)# monitor session 1 destination interface fa0/20

üîç Verifikasi


Switch# show monitor

Sekarang traffic dari server Switch A bisa dipantau di Switch B melalui port Wireshark.

‚úÖ Hasil yang Diharapkan

  • Traffic dari switch remote dapat dimonitor tanpa mengganggu operasional normal
  • RSPAN VLAN berhasil menghubungkan source dan destination antar switch
  • Monitoring terpusat di Switch B melalui port Wireshark

Lab 5 ‚Äì EtherChannel Mode On üîó

üìå Penjelasan Singkat

EtherChannel adalah teknik menggabungkan beberapa link fisik menjadi satu link logis untuk meningkatkan bandwidth dan redundansi.

Mode On ‚Üí membuat EtherChannel static, tanpa negosiasi protokol PAgP atau LACP.

Biasanya digunakan untuk menghubungkan switch ke switch atau switch ke server dengan multiple NIC.

üñ•Ô∏è Contoh Kasus

Dua switch (Switch A dan Switch B) memiliki 4 kabel gigabit. Tujuannya: gabungkan 4 link menjadi 1 logical link agar bandwidth 4 Gbps dan failover otomatis jika salah satu kabel putus.

‚öôÔ∏è Konfigurasi

1Ô∏è‚É£ Pilih port untuk digabung

Switch(config)# interface range gi0/1-4
Switch(config-if-range)# channel-group 1 mode on
2Ô∏è‚É£ Buat Port-Channel

Switch(config)# interface port-channel 1
Switch(config-if)# switchport mode trunk   # jika membawa VLAN
Switch(config-if)# switchport trunk allowed vlan 10,20,30
3Ô∏è‚É£ Verifikasi

Switch# show etherchannel summary
Switch# show interfaces port-channel

Output menunjukkan port-channel terbentuk, member aktif, dan trunk berjalan.

‚úÖ Hasil yang Diharapkan

  • Beberapa link fisik tergabung menjadi satu port-channel logis
  • Bandwidth gabungan meningkat sesuai jumlah link
  • Failover otomatis jika salah satu link putus
  • Trunk berjalan normal jika EtherChannel membawa VLAN

Lab 6 ‚Äì EtherChannel PAgP üîóü§ù

üìå Penjelasan Singkat

PAgP (Port Aggregation Protocol) adalah protokol Cisco proprietary untuk negosiasi EtherChannel secara otomatis.

PAgP memastikan link yang compatible bisa digabung tanpa konfigurasi statis.

Mode PAgP:

  • Auto ‚Üí pasif, menunggu partner memulai.
  • Desirable ‚Üí aktif mencoba membuat EtherChannel.

üñ•Ô∏è Contoh Kasus

Dua switch Cisco (Switch A dan Switch B) ingin membuat 4 link gigabit trunk menjadi satu EtherChannel secara otomatis. Menggunakan PAgP agar negosiasi otomatis, tidak perlu konfigurasi statis.

‚öôÔ∏è Konfigurasi

1Ô∏è‚É£ Konfigurasi PAgP di Switch A

Switch(config)# interface range gi0/1-4
Switch(config-if-range)# channel-group 1 mode desirable
Switch(config-if-range)# exit

Switch(config)# interface port-channel 1
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk allowed vlan 10,20,30
2Ô∏è‚É£ Konfigurasi PAgP di Switch B

Switch(config)# interface range gi0/1-4
Switch(config-if-range)# channel-group 1 mode auto
Switch(config-if-range)# exit

Switch(config)# interface port-channel 1
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk allowed vlan 10,20,30
3Ô∏è‚É£ Verifikasi

Switch# show etherchannel summary
Switch# show interfaces port-channel

EtherChannel terbentuk secara otomatis jika mode auto/desirable cocok, dan trunk berjalan normal.

‚úÖ Hasil yang Diharapkan

  • Link gigabit tergabung otomatis menjadi port-channel
  • Trunk VLAN berjalan melalui port-channel
  • Negosiasi PAgP berhasil tanpa konfigurasi statis

Lab 7 ‚Äì EtherChannel LACP üîó‚ö°

üìå Penjelasan Singkat

LACP (Link Aggregation Control Protocol) adalah standar IEEE 802.3ad untuk negosiasi EtherChannel otomatis.

LACP memungkinkan switch dari vendor berbeda bisa membentuk EtherChannel.

Dua switch (Switch A & Switch B) ingin menggabungkan 4 link gigabit menjadi EtherChannel trunk. Menggunakan LACP agar kompatibel dengan switch vendor lain dan negosiasi otomatis.

Switch(config-if-range)# exit

Switch(config)# interface port-channel 1
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk allowed vlan 10,20,30
2Ô∏è‚É£ Switch B (Passive)

Switch(config)# interface range gi0/1-4
Switch(config-if-range)# channel-group 1 mode passive
Switch(config-if-range)# exit

Switch(config)# interface port-channel 1
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk allowed vlan 10,20,30
3Ô∏è‚É£ Verifikasi

Switch# show etherchannel summary
Switch# show interfaces port-channel

EtherChannel terbentuk secara otomatis, trunk berjalan, dan kompatibel dengan perangkat vendor lain.

‚úÖ Hasil yang Diharapkan

  • Link gigabit tergabung menjadi port-channel secara otomatis
  • Trunk VLAN berjalan melalui port-channel
  • Negosiasi LACP berhasil dan kompatibel dengan vendor lain

Lab 8 ‚Äì EtherChannel Layer 3 üåêüîó

üìå Penjelasan Singkat

EtherChannel tidak hanya untuk Layer 2 (trunk), tapi juga bisa dibuat sebagai Layer 3 interface.

Biasanya digunakan untuk routing antar switch atau uplink ke router dengan agregasi link.

Bedanya: port-channel diberi IP address, dan tidak ada mode trunk (default L3).

üñ•Ô∏è Contoh Kasus

Dua switch Cisco ingin menggabungkan 4 link gigabit menjadi satu port-channel Layer 3. Tujuannya: meningkatkan bandwidth untuk routing antar switch, sekaligus redundancy.

‚öôÔ∏è Konfigurasi

1Ô∏è‚É£ Buat EtherChannel L3

Switch(config)# interface range gi0/1-4
Switch(config-if-range)# no switchport        # ubah ke Layer 3
Switch(config-if-range)# channel-group 1 mode on
2Ô∏è‚É£ Konfigurasi Port-Channel L3

Switch(config)# interface port-channel 1
Switch(config-if)# ip address 192.168.1.1 255.255.255.0
Switch(config-if)# no shutdown
3Ô∏è‚É£ Verifikasi

Switch# show etherchannel summary
Switch# show ip interface brief

Sekarang port-channel 1 bisa dipakai untuk routing antar switch. Jika satu link fisik down, traffic tetap berjalan di link lain.

‚úÖ Hasil yang Diharapkan

  • Link fisik tergabung menjadi port-channel Layer 3
  • Port-channel memiliki IP address dan bisa digunakan untuk routing
  • Redundancy aktif: jika satu link down, traffic tetap berjalan di link lain
  • Bandwidth gabungan meningkat sesuai jumlah link

Lab 9 ‚Äì STP Tuning: PortFast ‚ö°

üìå Penjelasan Singkat

STP (Spanning Tree Protocol) mencegah looping di jaringan Layer 2.

Biasanya, port STP butuh 30‚Äì50 detik untuk berpindah ke forwarding state.

PortFast: memaksa port langsung ke forwarding state saat switch start, cocok untuk port end-device seperti PC atau server.

Catatan: jangan aktifkan PortFast di port yang terhubung ke switch lain, bisa menyebabkan loop.

üñ•Ô∏è Contoh Kasus

Port Fa0/10 terhubung ke PC di VLAN 10. Tanpa PortFast, PC harus menunggu beberapa detik sebelum bisa komunikasi. Dengan PortFast, port langsung forwarding saat switch up ‚Üí boot & koneksi cepat.

‚öôÔ∏è Konfigurasi


Switch(config)# interface fa0/10
Switch(config-if)# spanning-tree portfast

üîç Verifikasi


Switch# show spanning-tree interface fa0/10

Output akan menunjukkan PortFast Enabled dan port langsung berada di forwarding state.

‚úÖ Hasil yang Diharapkan

  • Port end-device langsung forwarding saat switch up
  • Boot & koneksi perangkat lebih cepat
  • STP tetap mencegah loop di port lain

Lab 10 ‚Äì STP Tuning: UplinkFast ‚ö°üîÑ

üìå Penjelasan Singkat

UplinkFast digunakan pada switch akses (access switch) untuk mempercepat konvergensi STP ketika link uplink ke root bridge gagal.

Dengan UplinkFast, port yang menjadi cadangan (backup root port) segera menjadi root port, sehingga downtime minimal.

Hanya direkomendasikan untuk switch akses, jangan di core switch, karena bisa menyebabkan loop jika salah konfigurasi.

üñ•Ô∏è Contoh Kasus

Switch akses memiliki 2 uplink ke distribution switch. Jika uplink utama putus, tanpa UplinkFast butuh 30‚Äì50 detik untuk STP memilih port baru ‚Üí downtime. Dengan UplinkFast, port cadangan langsung forwarding ‚Üí downtime minimal.

‚öôÔ∏è Konfigurasi


Switch(config)# spanning-tree uplinkfast

üîç Verifikasi


Switch# show spanning-tree

Perhatikan root port dan backup port, waktu failover lebih cepat dibanding default STP.

‚úÖ Hasil yang Diharapkan

  • Switch akses cepat berkonvergensi saat uplink gagal
  • Downtime minimal, port cadangan langsung menjadi root port
  • STP tetap mencegah loop di jaringan lain

Lab 11 ‚Äì STP Tuning: BPDU Guard üõ°Ô∏è‚ö°

üìå Penjelasan Singkat

BPDU Guard adalah fitur proteksi STP untuk port yang terhubung ke end device.

Jika port yang di-enable BPDU Guard menerima BPDU (Bridge Protocol Data Unit), port akan shutdown otomatis.

Tujuan: mencegah switch tidak sah mengirim BPDU dan memengaruhi topologi STP.

Biasanya digabung dengan PortFast.

üñ•Ô∏è Contoh Kasus

Port Fa0/10 terhubung ke PC. Kita aktifkan PortFast dan BPDU Guard. Jika seseorang menghubungkan switch lain ke port ini, port langsung shutdown ‚Üí mencegah loop.

‚öôÔ∏è Konfigurasi


Switch(config)# interface fa0/10
Switch(config-if)# spanning-tree portfast
Switch(config-if)# spanning-tree bpduguard enable

üîç Verifikasi


Switch# show spanning-tree interface fa0/10 detail
Switch# show errdisable recovery

Jika port menerima BPDU, status menjadi err-disabled, dan perlu recovery untuk mengaktifkan kembali.

‚úÖ Hasil yang Diharapkan

  • Port end-device aman dari loop karena BPDU tidak sah
  • Port yang menerima BPDU otomatis shutdown
  • Digabungkan dengan PortFast untuk boot cepat tanpa risiko loop

Lab 12 ‚Äì STP Tuning: BPDU Filter üõ°Ô∏èüö´

üìå Penjelasan Singkat

BPDU Filter mencegah port mengirim dan/atau menerima BPDU.

Tujuan: membuat port seolah-olah tidak ikut STP, cocok untuk port end-device tertentu.

Perbedaan dengan BPDU Guard:

  • BPDU Guard ‚Üí shutdown port jika BPDU diterima.
  • BPDU Filter ‚Üí port tidak mengirim BPDU, bisa tetap forwarding.

Bisa diaktifkan globally atau per interface.

üñ•Ô∏è Contoh Kasus

Port Fa0/10 terhubung ke server yang tidak perlu ikut STP. Kita ingin port langsung forwarding dan tidak mengirim BPDU.

‚öôÔ∏è Konfigurasi

1Ô∏è‚É£ Per Interface

Switch(config)# interface fa0/10
Switch(config-if)# spanning-tree bpdufilter enable
2Ô∏è‚É£ Global Mode (opsional)

Switch(config)# spanning-tree portfast bpdufilter default

üîç Verifikasi


Switch# show spanning-tree interface fa0/10 detail

Port akan langsung forwarding, dan BPDU tidak dikirim.

‚úÖ Hasil yang Diharapkan

  • Port end-device langsung forwarding tanpa menunggu STP
  • BPDU tidak dikirim dari port tersebut
  • Port tetap aman dari loop karena tidak menerima BPDU

Lab 13 ‚Äì STP Tuning: Root Guard üõ°Ô∏èüëë

üìå Penjelasan Singkat

Root Guard mencegah switch lain menjadi root bridge pada port tertentu.

Jika port menerima BPDU dari switch dengan priority lebih rendah, port akan masuk status root-inconsistent dan tetap blocking.

Tujuan: menjaga root bridge tetap di lokasi yang diinginkan (biasanya core/distribution switch).

üñ•Ô∏è Contoh Kasus

Switch core (Switch A) adalah root bridge. Port yang menghubungkan ke switch akses (Switch B) ingin tidak membiarkan Switch B mengambil alih root. Dengan Root Guard, jika Switch B mencoba menjadi root, port akan diblock sementara.

‚öôÔ∏è Konfigurasi


Switch(config)# interface gi0/1
Switch(config-if)# spanning-tree guard root

üîç Verifikasi


Switch# show spanning-tree inconsistentports

Jika port menjadi root-inconsistent, STP tetap menjaga topologi jaringan stabil.

‚úÖ Hasil yang Diharapkan

  • Root bridge tetap di switch core/distribution yang diinginkan
  • Port yang menerima BPDU root tidak sah menjadi root-inconsistent
  • STP tetap mencegah loop dan menjaga stabilitas topologi

Lab 14 ‚Äì MSTP (Multiple Spanning Tree Protocol) üåêüï∏Ô∏è

üìå Penjelasan Singkat

MSTP (IEEE 802.1s) memungkinkan beberapa VLAN digabung ke dalam satu instance STP, sehingga lebih efisien daripada membuat STP terpisah untuk setiap VLAN.

Tujuan: mengurangi jumlah topologi STP, mempercepat konvergensi, dan mengoptimalkan penggunaan link.

üèóÔ∏è Struktur MSTP

  • MST Region ‚Üí kumpulan switch dengan konfigurasi MST sama.
  • MST Instance (MSTI) ‚Üí setiap instance bisa memetakan beberapa VLAN.
  • CIST (Common and Internal Spanning Tree) ‚Üí backbone antara region MST.

üñ•Ô∏è Contoh Kasus

Kantor memiliki 6 VLAN: 10‚Äì15. Tujuan konfigurasi:

  • VLAN 10‚Äì12 ‚Üí MST Instance 1
  • VLAN 13‚Äì15 ‚Üí MST Instance 2

Dengan MSTP, link cadangan bisa digunakan lebih optimal dibanding STP klasik.

‚öôÔ∏è Konfigurasi (Contoh Dasar)


Switch(config)# spanning-tree mode mst
Switch(config)# spanning-tree mst configuration
Switch(config-mst)# name MST-Region1
Switch(config-mst)# revision 1
Switch(config-mst)# instance 1 vlan 10-12
Switch(config-mst)# instance 2 vlan 13-15
Switch(config-mst)# exit

üîç Verifikasi


Switch# show spanning-tree mst
Switch# show spanning-tree mst detail

Output menampilkan MST instance, VLAN mapping, dan root bridge per instance.

‚úÖ Hasil yang Diharapkan

Lab 18 ‚Äì Port Security: Violation Action ‚ö†Ô∏èüîí

Lab 19 ‚Äì HSRP: Konfigurasi Dasar üîÑüõ°Ô∏è

üìå Penjelasan Singkat

HSRP (Hot Standby Router Protocol) adalah protokol Cisco untuk redundansi gateway.

  • Active router: menangani traffic
  • Standby router: mengambil alih jika active down

Tujuan: gateway IP virtual tetap tersedia, mencegah downtime jaringan.

üñ•Ô∏è Contoh Kasus

Jaringan VLAN 10 memiliki 2 router: R1 dan R2. Virtual IP 192.168.10.1 digunakan sebagai default gateway. R1 sebagai active, R2 sebagai standby.

‚öôÔ∏è Konfigurasi

Router R1 (Active)

R1(config)# interface gi0/0
R1(config-if)# ip address 192.168.10.2 255.255.255.0
R1(config-if)# standby 1 ip 192.168.10.1
R1(config-if)# standby 1 priority 110
R1(config-if)# standby 1 preempt
Router R2 (Standby)

R2(config)# interface gi0/0
R2(config-if)# ip address 192.168.10.3 255.255.255.0
R2(config-if)# standby 1 ip 192.168.10.1
R2(config-if)# standby 1 priority 100
R2(config-if)# standby 1 preempt

üîç Verifikasi


R1# show standby
R2# show standby

Virtual IP 192.168.10.1 tetap aktif. Jika R1 down ‚Üí R2 otomatis mengambil alih.

‚úÖ Hasil yang Diharapkan

  • Virtual IP selalu tersedia sebagai default gateway
  • Failover terjadi otomatis jika router active down
  • Downtime jaringan minimal

Lab 20 ‚Äì HSRP: Track Route üõ∞Ô∏èüîÑ

üìå Penjelasan Singkat

HSRP Tracking memungkinkan menurunkan priority router HSRP jika link tertentu down.

Router dengan priority lebih tinggi tetap active, tapi jika link down, priority turun ‚Üí standby router mengambil alih.

üñ•Ô∏è Contoh Kasus

‚öôÔ∏è Konfigurasi

Router R1

R1(config)# interface gi0/0
R1(config-if)# standby 1 ip 192.168.10.1
R1(config-if)# standby 1 priority 110
R1(config-if)# standby 1 preempt
R1(config-if)# standby 1 track gi0/1 20
Router R2

R2(config)# interface gi0/0
R2(config-if)# standby 1 ip 192.168.10.1
R2(config-if)# standby 1 priority 100
R2(config-if)# standby 1 preempt

üîç Verifikasi


R1# show standby
R2# show standby

Jika gi0/1 di R1 down ‚Üí priority R1 turun 20 ‚Üí R2 mengambil alih sebagai active.

‚úÖ Hasil yang Diharapkan

  • Failover HSRP berbasis status link, bukan hanya router mati
  • Router standby otomatis mengambil alih jika priority active turun
  • Virtual IP selalu tersedia, downtime minimal

IP SLA memonitor ketersediaan jalur atau host secara real-time. HSRP dapat digabung dengan IP SLA untuk failover otomatis jika jalur ke target down, bukan hanya interface down.

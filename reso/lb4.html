Load Balancing (NTH / PCC)

NTH / PCC (Per-Connection Classifier) membagi trafik berdasarkan urutan koneksi atau paket menggunakan hashing. Umum digunakan untuk multi-WAN agar sesi/klien tertentu diarahkan ke uplink berbeda secara merata.

  • Kelebihan: Mudah diimplementasikan, tidak memerlukan routing protocol, cocok untuk membagi beban antar-sesi/klien.
  • Keterbatasan: Tidak aware terhadap bandwidth realtime; hindari per-packet balancing agar tidak terjadi reordering TCP.
  • Gunakan Saat: Multi-WAN rumahan/kantor dengan beberapa koneksi internet, ingin pembagian koneksi yang konsisten per-user atau per-session.

CONTOH KASUS

  • Tujuan: Mengimplementasikan NTH Packet Routing untuk membagi trafik keluar secara bergantian (round-robin per paket) ke dua jalur ISP menggunakan MikroTik Router.
  • Topologi: Router R.NTH memiliki dua jalur keluar ke ISP melalui interface ether1 dan ether2, serta menyediakan DHCP untuk jaringan LAN via ether5.
  • Catatan:
    • Mangle digunakan untuk menandai koneksi dengan parameter nth sehingga paket dapat dialokasikan secara bergantian ke dua jalur.
    • Marking routing membuat jalur terpisah (jalur_1 dan jalur_2) untuk mengatur forwarding paket sesuai tanda.
    • Routing default mencakup kedua gateway agar terjadi load sharing antar ISP.
    • DHCP server memberikan IP kepada client di jaringan LAN.
    • NAT digunakan untuk mengakses internet melalui kedua ISP.
    • DNS diset ke Google DNS (8.8.8.8 & 8.8.4.4) agar client dapat resolve nama domain.

R.NTH


/ip pool
add name=dhcp_pool0 ranges=10.10.10.2                # Pool DHCP untuk client LAN

/ip dhcp-server
add address-pool=dhcp_pool0 disabled=no interface=ether5 name=dhcp1  # DHCP server di LAN

/ip address
add address=192.168.1.2/30 interface=ether1 network=192.168.1.0  # IP ke ISP1
add address=172.16.1.2/30 interface=ether2 network=172.16.1.0     # IP ke ISP2
add address=10.10.10.1/30 interface=ether5 network=10.10.10.0     # IP LAN gateway

/ip dhcp-server network
add address=10.10.10.0/30 gateway=10.10.10.1       # Default gateway DHCP client

/ip dns
set servers=8.8.8.8,8.8.4.4                         # DNS Google

/ip firewall mangle
add action=mark-connection chain=prerouting in-interface=ether5 new-connection-mark=koneksi_1 nth=2,1  
# Tandai paket ke jalur_1 secara bergantian (1 dari 2)
add action=mark-connection chain=prerouting in-interface=ether5 new-connection-mark=koneksi_2 nth=2,2  
# Tandai paket ke jalur_2 secara bergantian (2 dari 2)

add action=mark-routing chain=prerouting connection-mark=koneksi_1 in-interface=ether5 new-routing-mark=jalur_1 passthrough=no  
# Routing jalur_1
add action=mark-routing chain=prerouting connection-mark=koneksi_2 in-interface=ether5 new-routing-mark=jalur_2 passthrough=no  
# Routing jalur_2

/ip firewall nat
add action=masquerade chain=srcnat out-interface=ether1  # NAT keluar ISP1
add action=masquerade chain=srcnat out-interface=ether2  # NAT keluar ISP2

/ip route
add distance=1 gateway=192.168.1.1 routing-mark=jalur_1   # Route jalur_1
add distance=1 gateway=172.16.1.1 routing-mark=jalur_2    # Route jalur_2
add distance=1 gateway=172.16.1.1,192.168.1.1             # Default ECMP: load sharing

/system identity
set name=R.NTH                                           # Nama router

Tips Praktis

  • Gunakan per-connection (bukan per-packet) agar trafik TCP tetap stabil.
  • Pastikan IP range atau pengguna dibagi merata agar load balancing efektif.
  • Dokumentasikan penandaan koneksi (connection mark) dan routing mark untuk memudahkan troubleshooting.

Ringkasan

NTH / PCC membagi koneksi antar WAN berdasarkan hash koneksi atau alamat sumber, meningkatkan distribusi trafik tanpa perlu protokol routing. Cocok untuk multi-WAN di lingkungan kantor atau rumahan.

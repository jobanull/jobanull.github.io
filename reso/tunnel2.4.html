SSTP adalah protokol VPN Layer 3 yang berjalan di atas HTTPS (TCP 443). Menggunakan SSL/TLS untuk enkripsi, SSTP dapat melewati firewall dan NAT dengan mudah, serta aman untuk produksi.

  • Tujuan: Menghubungkan remote user atau site secara aman melalui internet.
  • VPN: Data dienkripsi dengan SSL/TLS, mendukung autentikasi sertifikat dan username/password.
  • Use Case: Remote access VPN, bypass firewall yang membatasi protokol VPN lainnya.
  • Kelebihan: Aman, berjalan di TCP 443 sehingga mudah melewati NAT/firewall, kompatibel dengan Windows, Linux, dan MikroTik.
  • Keterbatasan: Lebih lambat dibanding L2TP/IPsec atau OpenVPN UDP karena berjalan di TCP; butuh sertifikat SSL.

Tips Praktis

  • Gunakan sertifikat SSL yang valid untuk keamanan maksimal.
  • SSTP cocok untuk remote user yang sering berada di jaringan publik atau jaringan dengan NAT ketat.
  • Pastikan port TCP 443 terbuka di firewall/NAT agar klien dapat terkoneksi.

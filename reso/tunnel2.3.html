PPTP adalah protokol VPN Layer 2 yang menggunakan GRE untuk tunneling dan MPPE untuk enkripsi. Mudah dikonfigurasi, tetapi saat ini tidak direkomendasikan untuk produksi karena keamanan yang rendah.

  • Tujuan: Membuat koneksi VPN sederhana antar site atau untuk remote user.
  • VPN: Data dienkripsi dengan MPPE, termasuk autentikasi username/password.
  • Use Case: Lab/testing, pembelajaran, atau skenario legacy yang tidak membutuhkan keamanan tinggi.
  • Kelebihan: Konfigurasi cepat dan mudah, kompatibel dengan banyak platform.
  • Keterbatasan: Enkripsi lemah, mudah diserang, tidak cocok untuk data sensitif.

CONTOH KASUS

  • Menghubungkan dua router menggunakan tunnel VPN PPTP.
  • RB1 bertindak sebagai PPTP Server, RB2 sebagai PPTP Client.
  • Digunakan untuk menghubungkan network 192.168.1.0/30 dengan 192.168.2.0/30.

PPTP-RB1 (Server)


/ip pool
add name=pptp_pool ranges=10.10.10.2                         # Pool IP untuk PPTP client (RB2 akan mendapat IP ini)

/ppp profile
add local-address=10.10.10.1 name=pptp_profile \
    remote-address=pptp_pool                                 # Profil PPP: RB1 = 10.10.10.1, client ambil dari pool

/interface pptp-server server
set enabled=yes                                              # Aktifkan PPTP server (tanpa IPsec, enkripsi lemah)

/ip address
add address=192.168.190.100/24 interface=ether1              # IP WAN RB1
add address=192.168.1.1/30 interface=ether2                  # IP LAN RB1

/ip route
add distance=1 dst-address=192.168.2.0/30 gateway=10.10.10.2 # Routing ke LAN RB2 via IP PPTP client

/ppp secret
add name=pptp1 password=pptp1 profile=pptp_profile \
    service=pptp                                             # User & password autentikasi PPTP client

/system identity
set name=PPTP-R1                                             # Nama router

PPTP-RB2 (Client)


/interface pptp-client
add connect-to=192.168.190.100 disabled=no name=pptp-out1 \
    user=pptp1 password=pptp1                                # PPTP client ke IP WAN RB1 dengan kredensial

/ip address
add address=192.168.190.101/24 interface=ether1              # IP WAN RB2
add address=192.168.2.1/30 interface=ether2                  # IP LAN RB2

/ip dhcp-server network
add address=192.168.2.0/30 gateway=192.168.2.1               # DHCP network LAN RB2

/ip route
add distance=1 dst-address=192.168.1.0/30 gateway=10.10.10.1 # Routing ke LAN RB1 via IP PPTP server

/system identity
set name=PPTP-R2                                             # Nama router

Tips Praktis

  • Hanya gunakan PPTP untuk lab atau testing; jangan untuk produksi karena keamanan lemah.
  • Pastikan port TCP 1723 dan protokol GRE (47) dibuka di firewall/NAT.
  • Gunakan L2TP/IPsec atau OpenVPN untuk koneksi yang aman.

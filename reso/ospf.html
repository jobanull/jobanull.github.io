Open Shortest Path First (OSPF)

OSPF adalah protokol routing IGP (Interior Gateway Protocol) berbasis link-state yang digunakan untuk mendistribusikan informasi routing di dalam satu Autonomous System (AS). OSPF menggunakan algoritma Dijkstra (Shortest Path First) untuk menentukan jalur terbaik. Protokol ini mendukung jaringan skala besar, multi-area, dan konvergensi cepat.

Karakteristik OSPF

  • Termasuk dalam kategori link-state routing protocol.
  • Menggunakan protokol transport IP (protocol number 89).
  • Konvergensi cepat dibandingkan RIP.
  • Mendukung VLSM/CIDR.
  • Mendukung konsep area untuk segmentasi routing.

Istilah Penting

  • Router ID: Identitas unik router OSPF (format IP address).
  • Area: Segmentasi jaringan OSPF untuk mengurangi LSDB dan beban CPU.
  • Backbone Area (Area 0): Area inti yang menghubungkan semua area lain.
  • LSA (Link State Advertisement): Pesan yang menginformasikan status link.
  • LSDB (Link State Database): Database topologi jaringan di tiap router.
  • SPF Tree: Struktur pohon jalur terpendek hasil perhitungan algoritma Dijkstra.

Tipe Router di OSPF

  • Internal Router: Semua interface berada dalam satu area.
  • Backbone Router: Terhubung ke Area 0.
  • Area Border Router (ABR): Menghubungkan Area 0 dengan area lain.
  • Autonomous System Boundary Router (ASBR): Menghubungkan OSPF dengan protokol routing lain.

Proses Kerja OSPF

  1. Router membentuk hubungan tetangga (adjacency) dengan OSPF Hello packet.
  2. Menukar informasi LSA untuk membangun LSDB.
  3. Menghitung jalur terpendek menggunakan algoritma Dijkstra.
  4. Mengisi routing table dengan rute terbaik.

Tipe LSA Umum

  • Type 1: Router LSA – informasi link di dalam area.
  • Type 2: Network LSA – informasi jaringan multi-access.
  • Type 3: Summary LSA – ringkasan rute antar-area.
  • Type 5: AS External LSA – rute dari luar OSPF AS.

Keunggulan OSPF

  • Konvergensi cepat.
  • Mendukung load balancing dengan cost yang sama (ECMP).
  • Dapat dioptimalkan dengan pembagian area.
  • Mendukung autentikasi (plaintext dan MD5).

CONTOH KASUS

  • Tujuan: Mengimplementasikan OSPF multi-area pada jaringan MikroTik untuk mendistribusikan routing dinamis antar-router.
  • Topologi: Jaringan terdiri dari 7 router (R1-R7) dengan beberapa area OSPF:
    • Area backbone (0.0.0.0) menghubungkan R1, R2, R3
    • Area2 (2.2.2.2) menghubungkan R2, R6, R7
    • Area3 (3.3.3.3) menghubungkan R3, R4, R5

    • Setiap router memiliki IP unik untuk setiap interface yang terhubung ke jaringan point-to-point.
    • Interface ditambahkan ke OSPF sesuai area masing-masing.
    • Network mapping dilakukan agar OSPF mengetahui subnet yang termasuk dalam setiap area.
    • Router diberi identitas unik dengan /system identity agar mudah diidentifikasi.
    • R1 juga mengatur NAT dan default route ke gateway internet untuk koneksi keluar.
    • DNS di beberapa router diset ke Google DNS (8.8.8.8 & 8.8.4.4) agar client dapat resolve nama domain.

Ringkasan

OSPF adalah protokol routing IGP yang efisien, scalable, dan handal. Dengan dukungan multi-area, OSPF dapat digunakan pada jaringan enterprise hingga ISP untuk memastikan rute optimal dengan konvergensi cepat.

OSPF Virtual Link 📡

Pengertian OSPF Virtual Link

Virtual Link adalah tunnel logical dalam OSPF yang menghubungkan area non-backbone (Area selain Area 0) ke Area 0 (Backbone) melalui transit area.

OSPF mensyaratkan bahwa semua area harus terhubung ke Area 0, karena Area 0 adalah backbone tempat semua routing OSPF dikonsolidasikan. Kalau ada area yang terpisah dari Area 0 (misalnya karena desain topologi), maka Virtual Link dipakai untuk membuat jalur logical backbone.

Kapan Virtual Link Dibutuhkan?

  • Saat ada area yang tidak langsung terhubung ke Area 0.
  • Contoh topologi: Area 1 --- Area 2 --- Area 0 → Area 1 tidak langsung ke Area 0, maka Virtual Link diperlukan melalui Area 2.
  • Saat backbone Area 0 terputus (split), Virtual Link bisa dipakai untuk menjembatani backbone yang terpisah.

Karakteristik OSPF Virtual Link

  • Dibuat antara dua ABR (Area Border Router).
  • Transit area (area yang dilalui Virtual Link) tidak boleh Stub Area.
  • Virtual Link akan membuat logical adjacency seolah-olah router tersebut langsung terhubung ke Area 0.
  • Parameter yang dinegosiasikan mirip dengan link fisik (Hello/Dead Interval, Authentication, dll).

Contoh Topologi


[R1] --- Area 1 --- [R2] --- Area 2 --- [R3] --- Area 0

R2 terhubung ke Area 1 dan Area 2
R3 terhubung ke Area 2 dan Area 0

Supaya Area 1 bisa terkoneksi ke Area 0 → Buat Virtual Link antara R2 (ABR di Area 1-2)
dan R3 (ABR di Area 2-0) melalui Area 2.

Konfigurasi Virtual Link (Cisco IOS)

Misalkan:

  • R2 ABR → Router ID: 2.2.2.2
  • R3 ABR → Router ID: 3.3.3.3
  • Transit area = Area 2

Pada R2:


router ospf 1
 area 2 virtual-link 3.3.3.3

Pada R3:


router ospf 1
 area 2 virtual-link 2.2.2.2

Verifikasi


show ip ospf virtual-links

Jika sukses, statusnya akan UP dan adjacency ke Area 0 terbentuk.

Kelemahan Virtual Link

  • Membuat topologi lebih kompleks → sulit troubleshooting.
  • Mengurangi scalability (karena transit area tidak bisa stub).
  • Biasanya hanya solusi sementara → lebih baik desain ulang agar semua area langsung terkoneksi ke Area 0.

Ringkasan

  • Virtual Link = cara membuat koneksi logical ke Area 0 melalui transit area.
  • Hanya dibuat antara ABR dan transit area tidak boleh stub.
  • Gunakan sebagai solusi sementara; desain topologi ideal tetap seluruh area terhubung langsung ke Area 0.

OSPF over GRE Tunnel 🌐

Pengertian OSPF over GRE Tunnel

GRE (Generic Routing Encapsulation) adalah protokol tunneling yang bisa membawa berbagai protokol (termasuk OSPF) melalui jaringan IP yang sudah ada.

OSPF over GRE digunakan untuk menjalankan OSPF di jaringan yang tidak langsung terhubung atau melewati WAN/Internet.

Jadi, GRE membuat “virtual link fisik” untuk OSPF, tapi di atas IP publik/private network.

Kapan Digunakan?

  • Menghubungkan OSPF Area 0 antar lokasi yang terpisah secara fisik.
  • Menghubungkan non-backbone area yang tidak bisa langsung ke Area 0.
  • Ketika infrastruktur fisik tidak mendukung OSPF adjacency langsung, misalnya ISP MPLS atau internet publik.

Karakteristik OSPF over GRE

  • OSPF melihat GRE sebagai link point-to-point.
  • Bisa digunakan di atas IP publik tanpa harus mengubah desain OSPF inti.
  • Memungkinkan OSPF tetap berjalan normal, meskipun jalur fisik sebenarnya berbeda.
  • GRE tunnel bisa dikombinasikan dengan IPsec jika ingin keamanan.

Contoh Topologi


Site A (R1) ---------- Internet ---------- Site B (R2)
        | GRE Tunnel | 
        OSPF Area 0  

R1 dan R2 seolah-olah langsung terhubung ke Area 0 melalui tunnel.
OSPF adjacency terbentuk di atas GRE tunnel.

Konfigurasi OSPF over GRE (Cisco IOS)

Misalkan:

  • Tunnel IP: R1 = 10.10.10.1, R2 = 10.10.10.2
  • Tunnel source/destination: R1 = 192.168.1.1, R2 = 192.168.2.1

1️⃣ Konfigurasi Tunnel


R1
interface Tunnel0
 ip address 10.10.10.1 255.255.255.0
 tunnel source 192.168.1.1
 tunnel destination 192.168.2.1

R2
interface Tunnel0
 ip address 10.10.10.2 255.255.255.0
 tunnel source 192.168.2.1
 tunnel destination 192.168.1.1

2️⃣ Konfigurasi OSPF


R1
router ospf 1
 network 10.10.10.0 0.0.0.255 area 0

R2
router ospf 1
 network 10.10.10.0 0.0.0.255 area 0

Setelah konfigurasi, OSPF akan membentuk adjacency melalui Tunnel0 seolah-olah R1 dan R2 terhubung langsung.

Kelebihan OSPF over GRE

  • Bisa menembus jaringan yang berbeda subnet atau ISP.
  • Tidak tergantung pada topologi fisik.
  • Fleksibel, bisa dikombinasikan dengan keamanan (IPsec).

Kekurangan

  • Tambah overhead karena GRE encapsulation.
  • Latensi sedikit lebih tinggi dibanding link langsung.
  • Harus memastikan MTU tunnel cukup agar OSPF packet tidak fragment.

OSPF Standar Area 🟦

Pengertian OSPF Standar Area

Standar Area adalah tipe area OSPF default jika kita tidak menandainya sebagai khusus (Stub, Totally Stub, NSSA, dll).

Semua router di area ini mempunyai link ke backbone (Area 0) secara langsung atau melalui ABR.

Router dalam standar area dapat menerima semua jenis LSA (Link State Advertisement) dari OSPF:

  • LSA Type 1: Router LSA (internal area)
  • LSA Type 2: Network LSA (broadcast/multi-access)
  • LSA Type 3: Summary LSA (dari ABR)
  • LSA Type 4: ASBR Summary LSA
  • LSA Type 5: External LSA (dari ASBR)

Karakteristik Standar Area

  • Menerima semua jenis LSA → routing lengkap.
  • Tidak ada pembatasan update eksternal.
  • Bisa menjadi area backbone (Area 0) atau area non-backbone.
  • Biasanya digunakan di jaringan internal utama yang membutuhkan routing penuh.

Kelebihan

  • Mendapatkan informasi routing lengkap.
  • Fleksibel untuk pertumbuhan jaringan.
  • Cocok untuk area inti atau pusat distribusi.

Kekurangan

  • Bisa lebih boros resource pada router karena menyimpan semua LSA.
  • Lebih kompleks dibanding area khusus seperti Stub/NSSA.

Contoh Topologi


          Area 0 (Backbone)
        /        |        \
   Area 1      Area 2    Area 3
(Standard Area) 

Semua area non-backbone bisa tetap standar area jika ingin menerima routing penuh.
ABR akan meneruskan semua summary LSA ke area tersebut.

OSPF Stub Area 🌿

Pengertian OSPF Stub Area

Stub Area adalah area OSPF yang tidak menerima LSA Type 5 (External LSA) dari ASBR.

Tujuannya: mengurangi ukuran routing table dan menghemat resource router.

Semua paket eksternal (dari luar OSPF) diarahkan ke default route yang diberikan oleh ABR.

Karakteristik Stub Area

  • Tidak ada LSA Type 5 (External LSA).
  • Menerima LSA Type 1, 2, dan 3 saja.
  • ABR menyediakan default route (0.0.0.0) untuk area ini.
  • Tidak bisa menjadi Area 0 (Backbone).
  • Biasanya digunakan di area ujung (edge area) jaringan internal.

Keuntungan Stub Area

  • Mengurangi ukuran routing table.
  • Mengurangi penggunaan CPU dan memory pada router di area tersebut.
  • Cocok untuk area dengan banyak router tapi sedikit koneksi ke luar.

Kekurangan

  • Tidak bisa langsung mencapai external network selain default route.
  • Kurang fleksibel dibanding Standard Area.

Contoh Topologi


          Area 0 (Backbone)
                |
              ABR
                |
           Stub Area (Area 1)

Router di Stub Area akan menerima routing internal (LSA 1, 2) 
dan summary dari ABR (LSA 3),
tapi semua eksternal route (LSA 5) diganti default route.

Konfigurasi Stub Area (Cisco IOS)


router ospf 1
 area 1 stub

Secara otomatis, ABR akan menyebarkan default route ke area ini.

OSPF Totally Stub Area 🚫🌿

Pengertian OSPF Totally Stub Area

Totally Stub Area adalah area OSPF yang tidak menerima LSA Type 3 (Summary) maupun Type 5 (External) dari ABR.

Tujuannya: meminimalkan routing table di area ujung dengan hanya mengandalkan default route untuk semua tujuan di luar area.

Karakteristik Totally Stub Area

  • Tidak ada LSA Type 3 → tidak ada info tentang network di area lain.
  • Tidak ada LSA Type 5 → tidak ada info tentang network eksternal.
  • Semua paket keluar area diarahkan ke 0.0.0.0 yang diberikan ABR.
  • Tidak bisa menjadi Area 0 (Backbone).
  • Hanya menerima LSA Type 1 (Router LSA) dan LSA Type 2 (Network LSA) di dalam area itu sendiri.

Keuntungan Totally Stub Area

  • Routing table sangat kecil → hemat memori dan CPU.
  • Cocok untuk area edge dengan banyak router tapi hanya sedikit jaringan internal.

Kekurangan

  • Tidak tahu rute spesifik ke area lain → semua diarahkan ke default route.
  • Kurang fleksibel untuk koneksi langsung ke area lain atau eksternal.

Contoh Topologi


          Area 0 (Backbone)
                |
              ABR
                |
   Totally Stub Area (Area 1)

Router di Totally Stub Area hanya menyimpan routing internal area
dan default route ke ABR untuk semua tujuan lain.

Konfigurasi TSA (Cisco IOS)


router ospf 1
 area 1 stub no-summary

no-summary menandakan tidak menerima LSA Type 3 → menjadikannya Totally Stub. ABR akan menyebarkan satu default route untuk seluruh area ini.

OSPF NSSA (Not-So-Stubby Area) 🌐

Pengertian NSSA Area

NSSA (Not-So-Stubby Area) adalah area stub yang diperbolehkan menerima beberapa external routes dari ASBR internal tanpa harus menerima semua LSA Type 5 dari OSPF eksternal.

Digunakan untuk area ujung yang perlu mengiklankan beberapa network eksternal, tapi tetap ingin menghemat routing table.

LSA eksternal dikonversi menjadi Type 7 di dalam NSSA, dan diterjemahkan menjadi Type 5 saat keluar area oleh ABR.

Karakteristik NSSA

  • Tidak menerima LSA Type 5 dari OSPF eksternal (seperti Stub Area).
  • Bisa mengiklankan internal ASBR → LSA Type 7 diubah menjadi Type 5 saat diteruskan ke backbone.
  • Bisa menjadi Totally NSSA → tidak menerima LSA Type 3 juga (summary dari ABR).
  • Cocok untuk area edge yang butuh mengiklankan beberapa eksternal route tapi tetap kecil routing table.

Totally NSSA

  • Sama seperti NSSA tapi lebih ketat.
  • Tidak menerima LSA Type 3 (Summary) → seperti Totally Stub.
  • Hanya mengandalkan default route dari ABR untuk network lain di luar area.

Keuntungan NSSA / Totally NSSA

  • Routing table lebih kecil dibanding Standard Area.
  • Tetap bisa mengiklankan network eksternal lokal (misal: MPLS VPN, internet link).
  • Flexible untuk desain area edge yang butuh “kontrol” LSA.

Kekurangan

  • Konfigurasi lebih kompleks dibanding Stub/Totally Stub.
  • Perlu perhatian LSA Type 7 → 5 translation di ABR.

Contoh Topologi


          Area 0 (Backbone)
                |
              ABR
                |
            NSSA Area (Area 2)
          /                     \
     ASBR (eksternal)        Internal Routers

ASBR di NSSA mengiklankan eksternal route → LSA Type 7
→ diterjemahkan ABR ke Type 5 untuk backbone.
Router internal NSSA tetap hemat routing table.

Konfigurasi NSSA (Cisco IOS)


router ospf 1
 area 2 nssa

Konfigurasi Totally NSSA


router ospf 1
 area 2 nssa no-summary

no-summary → menandakan tidak menerima LSA Type 3, sama seperti Totally Stub.

OSPF External Route Type 1 (E1) 🌍

Pengertian E1

E1 adalah tipe rute eksternal OSPF di mana biaya total (metric) ke jaringan eksternal menggabungkan:

  • Cost internal OSPF dari router lokal ke ASBR (router yang memasukkan rute eksternal).
  • External cost yang diberikan ASBR itu sendiri.

Rumus total metric E1:

Total Cost = Cost ke ASBR + External Cost

Biasanya digunakan untuk load balancing ketika ada banyak jalur ke rute eksternal.

Karakteristik E1

  • Biaya internal OSPF dihitung sampai ke ASBR.
  • External metric (yang diiklankan ASBR) ditambahkan ke biaya internal.
  • Digunakan ketika ingin menghitung jalur terpendek ke jaringan eksternal.
  • Default OSPF eksternal di Cisco biasanya E2, tapi bisa diganti E1.

Perbedaan E1 vs E2

Fitur E1 E2
Biaya total Cost internal + external Hanya external cost
Pengaruh jalur internal Ada Tidak
Cocok untuk Banyak jalur ke ASBR Jalur tunggal ke ASBR
Default Cisco Tidak Ya

Contoh Topologi


R1 --(OSPF)-- R2 -- ASBR -- Internet

Jika tipe E1:
R1 menghitung cost R1 → R2 → ASBR + external cost.

Jika tipe E2:
R1 hanya menggunakan external cost dari ASBR,
jalur internal OSPF tidak diperhitungkan.

Konfigurasi E1 (Cisco IOS)


router ospf 1
 redistribute static subnets metric-type 1

metric-type 1 → mengiklankan sebagai E1.

Distribute-List di OSPF 🚦

Pengertian

Distribute-List digunakan untuk memfilter rute yang masuk ke routing table (RIB), bukan ke LSDB (Link-State Database).

Artinya: LSA tetap ada di LSDB, tetapi rute tertentu tidak akan dipasang ke routing table jika diblok oleh distribute-list.

👉 Jadi, distribute-list tidak menghentikan flooding LSA, hanya mencegah instalasi ke RIB.

Jenis Filtering

  • Inbound Filtering (paling umum) Mengontrol rute mana yang boleh masuk ke routing table dari LSDB. 
    distribute-list ACL_NUMBER in [interface]
distribute-list prefix PREFIX_NAME in
  • Outbound Filtering Hanya berlaku di redistribution (misalnya dari OSPF ke BGP). ❌ Tidak bisa digunakan untuk mencegah flooding OSPF LSA antar router.

Contoh Kasus


Topologi:
R1 --- OSPF --- R2

R1 mengiklankan 192.168.10.0/24 dan 192.168.20.0/24.
Tujuan: R2 hanya menerima 192.168.10.0/24.
Konfigurasi dengan ACL

ip access-list standard BLOCK_NET
 permit 192.168.10.0 0.0.0.255
!
router ospf 1
 distribute-list BLOCK_NET in
Konfigurasi dengan Prefix-List

ip prefix-list ALLOW_NET seq 5 permit 192.168.10.0/24
!
router ospf 1
 distribute-list prefix ALLOW_NET in

Verifikasi

  • show ip route ospf → hanya terlihat 192.168.10.0/24.
  • show ip ospf database → tetap terlihat dua LSA (10.0/24 & 20.0/24).

👉 Perbedaan inilah yang membedakan distribute-list dengan area filter atau prefix-suppression.

Ringkasan

  • Distribute-list di OSPF = filter ke routing table, bukan LSDB.
  • Bisa menggunakan ACL atau Prefix-list.
  • Cocok untuk “menyembunyikan” route dari proses forwarding, tanpa menghentikan LSA flooding.

OSPF Internal Route Summarization 🗂️

Pengertian

Internal Summarization di OSPF adalah proses meringkas beberapa network internal (LSA Type 1 & 2) menjadi summary address saat dibawa keluar dari area oleh ABR.

Disebut juga Inter-Area Route Summarization, karena hasil ringkasan dikirim ke area lain dalam bentuk LSA Type 3 (Summary LSA).

Kenapa Summarization Penting?

  • Mengurangi ukuran LSDB & routing table → lebih efisien.
  • Mengurangi flooding LSA antar area.
  • Membuat topologi lebih stabil → perubahan kecil di area tertentu tidak tersebar ke semua area.

Syarat Summarization Internal

  • Hanya dilakukan di ABR, bukan router internal area.
  • Hanya berlaku untuk network internal area.
  • Tidak berlaku untuk external route (itu menggunakan ASBR Summarization).

Contoh Kasus


Area 1 memiliki subnet:
10.1.1.0/24
10.1.2.0/24
10.1.3.0/24
10.1.4.0/24

Tanpa summarization:
→ Semua /24 diiklankan ke Area 0.

Dengan summarization:
→ Bisa diringkas menjadi 10.1.0.0/16 (atau 10.1.0.0/22).

Konfigurasi (Cisco IOS)


router ospf 1
 area 1 range 10.1.0.0 255.255.0.0

area 1 → area asal network internal
range 10.1.0.0 255.255.0.0 → summary prefix

Verifikasi


show ip route ospf
show ip ospf database summary

Router di area lain hanya melihat summary 10.1.0.0/16. Subnet detail tetap ada di Area 1, tapi tidak dibawa keluar.

Ringkasan

  • Summarization Internal dilakukan di ABR dengan area range.
  • Meringkas internal routes (LSA 1 & 2) menjadi Summary LSA (Type 3).
  • Tujuan: mengurangi routing table & meningkatkan stabilitas OSPF.

OSPF External Route Summarization 🌐

Pengertian

External Summarization adalah proses meringkas beberapa rute eksternal (hasil redistribusi ke OSPF → LSA Type 5 atau LSA Type 7) menjadi satu summary route. Dilakukan di ASBR (Autonomous System Boundary Router), bukan di ABR.

Rute eksternal bisa berasal dari static route, RIP, EIGRP, BGP, atau protokol lain yang di-redistribusi ke OSPF.

Perbedaan dengan Internal Summarization

Fitur Internal Summarization External Summarization
Dilakukan di ABR ASBR
Rute yang diringkas Internal OSPF (LSA 1 & 2) External OSPF (LSA 5 atau 7)
Perintah Cisco IOS area range summary-address
LSA yang dihasilkan Type 3 (Summary LSA) Type 5 (External LSA)

Contoh Kasus


ASBR memiliki static route:
ip route 172.16.1.0 255.255.255.0 Null0
ip route 172.16.2.0 255.255.255.0 Null0
ip route 172.16.3.0 255.255.255.0 Null0
ip route 172.16.4.0 255.255.255.0 Null0

Tanpa summarization:
→ Semua /24 akan diiklankan sebagai LSA 5.

Dengan summarization:
→ Cukup satu entry saja: 172.16.0.0/16

Konfigurasi (Cisco IOS)


router ospf 1
 redistribute static subnets
 summary-address 172.16.0.0 255.255.0.0

redistribute static subnets → memasukkan static route ke OSPF.
summary-address 172.16.0.0 255.255.0.0 → meringkas semua /24 menjadi satu summary /16.

Verifikasi


show ip route ospf
show ip ospf database external

Harusnya hanya terlihat satu entry 172.16.0.0/16 sebagai external route.

Ringkasan

  • External Summarization = meringkas rute eksternal (hasil redistribusi).
  • Dilakukan di ASBR dengan summary-address.
  • Menghasilkan External LSA (Type 5 atau 7 → 5).
  • Membuat routing table lebih ringkas, stabil, dan hemat resource.

Redistribution Default Route di OSPF ✅

Pengertian

Dalam OSPF, kita bisa memasukkan default route (0.0.0.0/0) ke dalam domain OSPF supaya semua router tahu jalur keluar menuju Internet atau upstream router. Default route biasanya dibuat manual di ASBR atau ABR, lalu di-redistribute ke OSPF agar tersebar ke seluruh area.

Cara Membuat Default Route di OSPF

  1. Menggunakan default-information originate

    Paling umum digunakan. Router akan menyebarkan default route jika router tersebut punya default route di routing table.

    
router ospf 1
 default-information originate

    ➡️ OSPF akan mengiklankan 0.0.0.0/0 ke seluruh domain OSPF. Syarat: router harus sudah punya default route (misalnya static ke ISP).

  2. Menggunakan default-information originate always

    Dipakai kalau router tidak punya default route di routing table, tapi tetap ingin mengiklankan default route.

    
router ospf 1
 default-information originate always
  3. Menggunakan redistribute static

    Kita membuat default static route manual, lalu redistribusikan ke OSPF.

    
ip route 0.0.0.0 0.0.0.0 <next-hop-ip>
!
router ospf 1
 redistribute static subnets

    Cara ini lebih fleksibel, tetapi default route akan dianggap sebagai external route (Type 5 LSA).

Jenis LSA yang Digunakan

  • Default route diiklankan sebagai LSA Type 5 (External LSA).
  • Jika disebarkan di NSSA area → muncul sebagai Type 7, lalu diterjemahkan menjadi Type 5 di ABR.

Contoh Topologi


   [ISP]
     |
  (R1 - ASBR)
     |
   OSPF Area 0 ---- R2 ---- R3

R1 punya default route ke ISP:
ip route 0.0.0.0 0.0.0.0 200.1.1.1

Dengan 'default-information originate', 
R2 & R3 akan dapat default route via OSPF.

Verifikasi


show ip route ospf

Akan terlihat entry seperti: O*E2 0.0.0.0/0 [110/1] via <next-hop> E2 = External Type 2 (default). Bisa juga diubah menjadi E1 agar biaya internal ikut dihitung.

Ringkasan

  • default-information originate → cara umum menyebarkan default route di OSPF.
  • Opsi always → tetap mengiklankan default meski tidak ada default di RIB.
  • redistribute static → menghasilkan external LSA.
  • Default route muncul sebagai E1 atau E2 external route di OSPF.

Redistribution – Static ke OSPF 🚀

Pengertian

Redistribution Static = proses memasukkan static route yang ada di router ke dalam domain OSPF. Static route yang diredistribusikan akan menjadi External LSA (Type 5) jika dimasukkan oleh ASBR. Jika dilakukan di NSSA area → route menjadi LSA Type 7, lalu diterjemahkan ke Type 5 oleh ABR.

Kenapa Redistribute Static?

  • Mengiklankan default route (0.0.0.0/0) ke OSPF domain.
  • Mengiklankan network eksternal yang hanya diketahui via static (contoh: ISP, DMZ).
  • Integrasi rute sementara / policy-based routing ke OSPF.

Contoh Topologi


   [ISP: 200.1.1.1]
        |
       R1 (ASBR)
        |
     OSPF Area 0

Static route di R1:
ip route 0.0.0.0 0.0.0.0 200.1.1.1
ip route 172.16.10.0 255.255.255.0 200.1.1.2

Kita ingin supaya semua router OSPF tahu rute ini.

Konfigurasi Cisco IOS


router ospf 1
 redistribute static subnets

redistribute static → memasukkan static route ke OSPF.
subnets → wajib agar static route dengan subnet mask non-classful ikut terdistribusi.

Menentukan Metric Type (E1 vs E2)

  • Default → E2 (External Type 2).
  • Kalau mau E1 (cost internal ikut dihitung):

router ospf 1
 redistribute static subnets metric-type 1

Menambahkan Biaya (Metric)

Bisa override metric default (20) dengan manual:


router ospf 1
 redistribute static subnets metric 50

Filtering Static Route

Kalau hanya ingin memasukkan route tertentu:


ip prefix-list ONLY_DEFAULT seq 5 permit 0.0.0.0/0
!
router ospf 1
 redistribute static subnets route-map ONLY_DEFAULT

Verifikasi


show ip route ospf
show ip ospf database external

Harus terlihat static route sebagai O E2 (default) atau O E1 kalau pakai metric-type 1.

Ringkasan

  • Redistribute Static → OSPF = menjadikan static route sebagai External LSA.
  • Default = E2, bisa diubah ke E1.
  • Gunakan subnets agar semua static non-classful ikut terbawa.
  • Bisa difilter dengan prefix-list / route-map agar tidak semua static masuk OSPF.

Redistribution – Connected ke OSPF 🚀

Pengertian

Redistribution Connected = memasukkan semua connected interface router ke dalam OSPF sebagai External Route (LSA Type 5). Berbeda dengan network statement yang mengaktifkan adjacency di interface, redistribute connected hanya memasukkan prefix connected ke LSDB tanpa peduli adjacency.

Kenapa Redistribute Connected?

  • Mengiklankan interface yang tidak ingin ikut adjacency, tapi subnetnya perlu diketahui OSPF (contoh: loopback, management VLAN).
  • Mengiklankan link eksternal (misalnya ke ISP) tanpa perlu network statement.
  • Mengiklankan default route kalau dibuat sebagai connected (contoh: via Null0).

Contoh Kasus


Interface Router:
Loopback0:    10.10.10.1/32
Ethernet0/0:  192.168.1.1/24 (OSPF enabled)
Ethernet0/1:  200.1.1.1/30 (ISP link, tidak mau adjacency OSPF)

Config tanpa redistribute:
router ospf 1
 network 192.168.1.0 0.0.0.255 area 0
→ Hanya 192.168.1.0/24 diiklankan.

Kalau mau 10.10.10.1/32 & 200.1.1.0/30 ikut masuk OSPF:
router ospf 1
 redistribute connected subnets

Konfigurasi Cisco IOS


router ospf 1
 redistribute connected subnets
🔹 Opsi tambahan:
  • Menentukan metric:

router ospf 1
 redistribute connected subnets metric 50
  • Mengubah tipe external (default E2 → E1):

router ospf 1
 redistribute connected subnets metric-type 1
  • Membatasi interface tertentu (pakai prefix-list & route-map):

ip prefix-list ONLY_LOOPBACK seq 5 permit 10.10.10.1/32
!
route-map REDIST-CONN permit 10
 match ip address prefix-list ONLY_LOOPBACK
!
router ospf 1
 redistribute connected subnets route-map REDIST-CONN

Verifikasi


show ip route ospf
show ip ospf database external

Harusnya terlihat route connected masuk sebagai O E2 (default) atau O E1 kalau metric-type 1.

Ringkasan

  • Redistribute Connected → memasukkan semua connected interface ke OSPF.
  • Default → External LSA Type 5 (E2).
  • Berguna untuk loopback, manajemen, atau link eksternal.
  • Bisa difilter agar hanya interface tertentu yang ikut.

Redistribution – RIP ke OSPF 🚀

Pengertian

Redistribution RIP → OSPF artinya route yang didapat (atau dibuat) oleh RIP akan dimasukkan ke domain OSPF. Router yang melakukan redistribusi berperan sebagai ASBR (Autonomous System Boundary Router). Hasil redistribusi: route RIP akan muncul di OSPF sebagai External Route (LSA Type 5). Kalau redistribusi dilakukan di NSSA Area, maka route masuk sebagai Type 7 LSA lalu diterjemahkan jadi Type 5 oleh ABR.

Kenapa Redistribute RIP ke OSPF?

  • Transisi jaringan dari RIP ke OSPF (migrasi protokol lama ke baru).
  • Integrasi network kecil (pakai RIP) ke backbone OSPF.
  • Supaya semua router OSPF tahu rute yang hanya diketahui oleh domain RIP.

Contoh Topologi


   RIP domain
   (10.10.0.0/16)
      |
     R1 (ASBR)
      |
   OSPF Area 0

R1 menjalankan RIP untuk LAN lama (10.10.0.0/16).
R1 juga menjalankan OSPF sebagai backbone.
Agar router OSPF tahu jalur ke 10.10.0.0/16,
perlu redistribute RIP ke OSPF.

Konfigurasi Cisco IOS


router ospf 1
 redistribute rip subnets

Penjelasan:
subnets → wajib, agar semua subnet RIP (bukan hanya classful) ikut terdistribusi.
Default metric = 20, metric-type = E2 (external type 2).

Opsi Tambahan

  • Menentukan metric manual:

router ospf 1
 redistribute rip subnets metric 50
  • Mengubah menjadi Type E1 (biar metric internal ikut dihitung):

router ospf 1
 redistribute rip subnets metric-type 1
  • Filtering route RIP tertentu:

ip prefix-list ONLY_RIP_NET seq 5 permit 10.10.0.0/16
!
route-map RIP_TO_OSPF permit 10
 match ip address prefix-list ONLY_RIP_NET
!
router ospf 1
 redistribute rip subnets route-map RIP_TO_OSPF

Verifikasi


show ip route ospf
show ip ospf database external

Hasil: route RIP akan muncul di OSPF dengan tanda O E2 (default). Kalau metric-type 1 digunakan, akan muncul sebagai O E1.

Ringkasan

  • Redistribute RIP → OSPF = memasukkan route dari domain RIP ke domain OSPF.
  • Default: External Type 5 LSA (E2), metric 20.
  • Bisa diubah ke E1 atau set metric manual.
  • Bisa difilter supaya hanya route tertentu dari RIP yang diiklankan ke OSPF.

Redistribution – EIGRP ke OSPF 🚀

Pengertian

Redistribution EIGRP → OSPF = semua route yang ada di domain EIGRP akan di-inject ke dalam domain OSPF. Router yang melakukan redistribusi akan menjadi ASBR (Autonomous System Boundary Router). Route EIGRP di OSPF akan muncul sebagai External Route (LSA Type 5). Kalau redistribusi dilakukan di NSSA Area, maka route masuk sebagai Type 7 LSA dulu lalu diterjemahkan menjadi Type 5 oleh ABR.

Kenapa Redistribute EIGRP ke OSPF?

  • Transisi/migrasi jaringan dari EIGRP ke OSPF.
  • Interkoneksi antar-domain IGP berbeda.
  • Supaya backbone OSPF tetap tahu jalur menuju jaringan EIGRP.

Contoh Topologi


   [EIGRP Domain]
   172.16.0.0/16
        |
       R1 (ASBR)
        |
   [OSPF Area 0]

R1 menjalankan EIGRP AS 100 untuk LAN.
R1 juga menjalankan OSPF Area 0 ke backbone.
Agar semua router OSPF tahu rute 172.16.0.0/16, perlu redistribusi.

Konfigurasi Cisco IOS


router ospf 1
 redistribute eigrp 100 subnets

Penjelasan:
eigrp 100 = nomor autonomous-system EIGRP yang diredistribusi.
subnets = wajib, supaya semua prefix (termasuk non-classful) ikut disebarkan.
Default metric OSPF external = 20, tipe E2.

Opsi Tambahan

  • Menentukan metric manual:

router ospf 1
 redistribute eigrp 100 subnets metric 50
  • Mengubah metric-type (E1 vs E2):

router ospf 1
 redistribute eigrp 100 subnets metric-type 1

Default → E2 (hanya cost eksternal). Bisa diubah ke E1 agar biaya internal OSPF ikut dihitung.

  • Filtering route (hanya route tertentu):

ip prefix-list ONLY_EIGRP_NET seq 5 permit 172.16.0.0/16
!
route-map EIGRP_TO_OSPF permit 10
 match ip address prefix-list ONLY_EIGRP_NET
!
router ospf 1
 redistribute eigrp 100 subnets route-map EIGRP_TO_OSPF

Verifikasi


show ip route ospf
show ip ospf database external

🔎 Hasil: route dari EIGRP muncul sebagai O E2 (default). Kalau metric-type 1 digunakan → akan muncul sebagai O E1. Bisa dicek juga LSDB apakah Type 5 LSA muncul sesuai prefix yang di-redistribute.

Ringkasan

  • Redistribution EIGRP → OSPF menjadikan router sebagai ASBR.
  • Default: External LSA Type 5 (E2), metric 20.
  • Bisa diubah ke E1 atau metric lain sesuai kebutuhan.
  • Filtering dengan route-map/prefix-list penting agar tidak semua route EIGRP masuk OSPF.

OSPF Pindah Jalur (Path Selection) 🚦

Pengertian

OSPF adalah protokol link-state yang menentukan jalur terbaik berdasarkan cost. Cost dihitung dari bandwidth interface, dengan rumus default Cisco: 

Cost = Reference Bandwidth (100 Mbps) / Bandwidth Interface (Mbps)

⚠️ Maksimum cost = 65.535.
Jika ada lebih dari satu jalur:

  • OSPF pilih jalur dengan cost terkecil.
  • Kalau cost sama → Equal Cost Load Balancing (ECMP).
  • Kalau cost berbeda → hanya jalur dengan cost terendah dipakai, lainnya jadi backup.

Contoh Topologi


R1 ----(100 Mbps)---- R2
 |                    |
(10 Mbps)          (100 Mbps)
 |                    |
R3 --------------------

Dari R1 ke R3 ada 2 jalur:
- R1–R2–R3 via link 100 Mbps
- R1–R3 langsung via link 10 Mbps

Perhitungan Cost:
100 Mbps → cost = 1
10 Mbps  → cost = 10

➡️ OSPF akan pilih jalur lewat R1–R2–R3 (100 Mbps).

Manipulasi Jalur di OSPF

  1. Mengubah Reference Bandwidth
    Default reference = 100 Mbps (tidak cocok di jaringan modern). Bisa diubah: 
    
router ospf 1
 auto-cost reference-bandwidth 10000
    👉 Reference sekarang 10 Gbps.
  2. Mengubah Cost di Interface
    Bisa pakai manual cost, misalnya untuk backup link: 
    
interface GigabitEthernet0/0
 ip ospf cost 50
  3. Mengatur Load Balancing
    Kalau cost dibuat sama di dua jalur: 
    
interface Gi0/0
 ip ospf cost 10
!
interface Gi0/1
 ip ospf cost 10
    ➡️ OSPF akan load balance (ECMP).
  4. Unequal Path (Backup)
    OSPF tidak mendukung unequal cost load balancing (beda dengan EIGRP). Kalau cost beda → hanya cost terendah yang dipakai, lainnya standby.

Verifikasi


show ip route ospf
show ip ospf interface
show ip ospf database

Bisa dipakai untuk cek metric dan jalur aktif.

Ringkasan

  • OSPF memilih jalur berdasarkan cost terkecil.
  • Default cost = 100 Mbps / bandwidth interface.
  • Bisa diubah lewat ip ospf cost atau auto-cost reference-bandwidth.
  • ECMP: kalau cost sama → load balancing.
  • Unequal cost: tidak didukung, hanya link dengan cost terendah yang dipakai.

OSPF Authentication 🔐

Pengertian

OSPF Authentication adalah mekanisme untuk mengamankan pertukaran LSA antar-router OSPF. Tujuan utamanya adalah mencegah router asing atau attacker masuk ke domain OSPF dan merusak Link-State Database (LSDB). Authentication bisa diterapkan di level area atau interface.

Jenis Authentication di OSPF

  1. Null Authentication (Default)
    Tidak ada autentikasi. Semua router OSPF bisa bertukar LSA selama area sama. ➡️ Default mode OSPF.
  2. Plaintext Authentication
    Password dikirim dalam bentuk clear text (tidak dienkripsi). Lemah secara keamanan. 
    
interface GigabitEthernet0/0
 ip ospf authentication
 ip ospf authentication-key cisco123
  3. MD5 Authentication (Message Digest 5)
    Password tidak dikirim langsung, tapi di-hash MD5. Umum dipakai di jaringan enterprise. 
    
interface GigabitEthernet0/0
 ip ospf authentication message-digest
 ip ospf message-digest-key 1 md5 cisco123
    👉 1 = key-id (untuk rotasi key), cisco123 = shared secret.
  4. HMAC-SHA Authentication (OSPFv3 / IPv6)
    Di OSPFv3, authentication menggunakan IPsec (AH/ESP) dengan HMAC-SHA. Lebih aman daripada MD5.

Konfigurasi di Area vs Interface

  • Area Authentication
    
router ospf 1
 area 0 authentication message-digest
    ➡️ Semua interface di Area 0 wajib pakai MD5.
  • Interface Authentication
    
interface Gi0/0
 ip ospf authentication message-digest
 ip ospf message-digest-key 1 md5 rahasia123
    ➡️ Lebih granular, bisa beda antar-interface.

Syarat Sukses Authentication

  • Semua router di link harus pakai tipe autentikasi yang sama.
  • Kalau satu pakai MD5, satu pakai plaintext → adjacency gagal.
  • Password (key) harus sama persis.
  • Untuk MD5, key-id boleh beda, asal password sama.

Verifikasi


show ip ospf neighbor
show ip ospf interface

Jika authentication gagal → OSPF adjacency tidak terbentuk. Error yang sering muncul: Mismatched Authentication Type atau Key not valid.

Ringkasan

  • OSPF mendukung Null, Plaintext, MD5, dan OSPFv3 mendukung IPsec/HMAC-SHA.
  • MD5 Authentication adalah pilihan paling umum (aman & simpel).
  • Bisa diterapkan di area atau interface.
  • Semua router di link harus cocok tipe & password-nya.

OSPF dalam VRF Lite 👍

Konsep OSPF di VRF Lite

Saat membuat VRF Lite, otomatis setiap VRF punya routing table sendiri. Jika OSPF dijalankan dalam VRF, maka OSPF juga berjalan terpisah per VRF. ➡️ OSPF instance VRF-A tidak akan tahu jalur di VRF-B. ➡️ Bisa ada beberapa OSPF process dengan vrf <nama_vrf>.

Contoh Kasus


R1
+ Gi0/0 → VRF CUST_A → OSPF process untuk Customer A
+ Gi0/1 → VRF CUST_B → OSPF process untuk Customer B

Customer A & B bisa pakai IP yang sama (overlap). Keduanya butuh routing dinamis dengan OSPF. Solusi → jalankan OSPF per VRF.

Konfigurasi Cisco IOS

  1. Buat VRF 
    
ip vrf CUST_A
 rd 1:100
!
ip vrf CUST_B
 rd 1:200
  2. Masukkan interface ke VRF 
    
interface Gi0/0
 ip vrf forwarding CUST_A
 ip address 10.1.1.1 255.255.255.0
!
interface Gi0/1
 ip vrf forwarding CUST_B
 ip address 10.1.1.1 255.255.255.0
    👉 Kedua interface bisa pakai IP sama karena terisolasi di VRF berbeda.
  3. Jalankan OSPF di masing-masing VRF 
    
router ospf 10 vrf CUST_A
 router-id 1.1.1.1
 network 10.1.1.0 0.0.0.255 area 0
!
router ospf 20 vrf CUST_B
 router-id 2.2.2.2
 network 10.1.1.0 0.0.0.255 area 0
    ➡️ Bedanya dengan OSPF biasa: ada keyword vrf <nama_vrf>.

Routing Table


show ip route vrf CUST_A
show ip route vrf CUST_B

Routing OSPF VRF-A tidak muncul di VRF-B, masing-masing punya database sendiri. Bisa dicek dengan show ip ospf vrf CUST_A database.

Integrasi antar-VRF (Opsional)

Jika perlu komunikasi antar VRF, opsi yang bisa dipakai:

  • Route leaking (static / import-export).
  • BGP VPNv4 (kalau sudah MPLS).

ip route vrf CUST_A 192.168.20.0 255.255.255.0 10.1.1.2 global
ip route vrf CUST_B 192.168.10.0 255.255.255.0 10.1.1.2 global

Ringkasan

  • OSPF VRF Lite = OSPF per VRF, tiap customer/tenant punya OSPF sendiri.

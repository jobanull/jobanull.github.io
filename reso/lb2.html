Load Balancing (ECMP)

ECMP (Equal-Cost Multi-Path) adalah teknik load balancing pada layer 3, yang membagi trafik ke beberapa jalur dengan biaya (cost) yang sama. Ideal untuk jaringan dengan beberapa gateway atau link ke tujuan yang sama.

  • Kelebihan: Membagi beban trafik secara merata antar-gateway/link, meningkatkan throughput, dan memanfaatkan jalur redundan.
  • Keterbatasan: Satu flow biasanya tetap pada satu jalur; perlu routing protocol atau static route dengan cost sama.
  • Gunakan Saat: Multi-WAN, backbone dengan banyak jalur, link ke ISP/core yang setara.

CONTOH KASUS

  • Tujuan: Mengimplementasikan ECMP (Equal-Cost Multi-Path) untuk membagi trafik keluar ke dua ISP sekaligus menggunakan MikroTik Router.
  • Topologi: Router R.ECMP memiliki dua jalur keluar ke ISP berbeda melalui interface ether1 dan ether2, serta menyediakan DHCP untuk jaringan LAN via ether5.
  • Catatan:
    • Marking connection & routing digunakan agar paket keluar melalui ISP yang sesuai.
    • Routing default dibuat dengan dua gateway agar terjadi load balancing secara ECMP.
    • DHCP server memberikan IP kepada client di jaringan LAN.
    • DNS diset ke Google DNS (8.8.8.8 & 8.8.4.4) agar client dapat resolve nama domain.
    • NAT digunakan untuk mengakses internet melalui kedua ISP.

R.ECMP


/ip pool
add name=dhcp_pool0 ranges=10.10.10.2                # Pool DHCP untuk client LAN

/ip dhcp-server
add address-pool=dhcp_pool0 disabled=no interface=ether5 name=dhcp1  # DHCP server di LAN

/ip address
add address=192.168.1.2/30 interface=ether1 network=192.168.1.0  # IP ke ISP1
add address=172.16.1.2/30 interface=ether2 network=172.16.1.0     # IP ke ISP2
add address=10.10.10.1/30 interface=ether5 network=10.10.10.0     # IP LAN gateway

/ip dhcp-server network
add address=10.10.10.0/30 gateway=10.10.10.1       # Default gateway DHCP client

/ip dns
set servers=8.8.8.8,8.8.4.4                         # DNS Google

/ip firewall mangle
add action=mark-connection chain=input in-interface=ether1 new-connection-mark=ISP1  # Tandai koneksi masuk ISP1
add action=mark-connection chain=input in-interface=ether2 new-connection-mark=ISP2  # Tandai koneksi masuk ISP2
add action=mark-routing chain=output connection-mark=ISP1 new-routing-mark=ISP1       # Routing berdasarkan tanda ISP1
add action=mark-routing chain=output connection-mark=ISP2 new-routing-mark=ISP2       # Routing berdasarkan tanda ISP2

/ip firewall nat
add action=masquerade chain=srcnat out-interface=ether1  # NAT ke ISP1
add action=masquerade chain=srcnat out-interface=ether2  # NAT ke ISP2

/ip route
add check-gateway=ping distance=1 gateway=192.168.1.1 routing-mark=ISP1  # Default route ISP1
add check-gateway=ping distance=1 gateway=172.16.1.1 routing-mark=ISP2   # Default route ISP2
add distance=1 gateway=192.168.1.1,172.16.1.1                        # ECMP: kedua gateway untuk load balancing

/system identity
set name=R.ECMP                                           # Nama router

Tips Praktis

  • Pastikan semua gateway/link yang digunakan ECMP memiliki cost yang sama.
  • Per-connection balancing lebih stabil untuk trafik TCP, hindari per-packet agar tidak terjadi reordering.
  • Monitor kesehatan jalur dengan check-gateway atau IP SLA untuk failover otomatis jika salah satu link putus.

Ringkasan

ECMP membagi trafik IP secara merata ke beberapa jalur dengan cost sama, meningkatkan throughput dan memanfaatkan jalur redundan. Cocok untuk multi-WAN atau backbone dengan banyak link setara. Satu flow biasanya tetap pada jalur yang sama.

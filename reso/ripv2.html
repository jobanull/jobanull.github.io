Routing Information Protocol (RIP & RIPv2)

RIP adalah protokol routing IGP (Interior Gateway Protocol) berbasis distance vector yang menggunakan hop count sebagai metrik untuk menentukan jalur terbaik. Setiap router RIP secara berkala mengirimkan tabel routing lengkap ke tetangganya setiap 30 detik.

Karakteristik RIP

  • Menggunakan hop count sebagai metrik (maksimal 15 hop, 16 = tidak terjangkau).
  • Mengirim update routing setiap 30 detik.
  • Menggunakan algoritma Bellman-Ford.
  • Protokol transport: UDP port 520.
  • Tidak cocok untuk jaringan besar karena keterbatasan hop count.

Perbedaan RIP vs RIPv2

Fitur RIP (v1) RIPv2
Tipe Routing Classful (tidak kirim subnet mask) Classless (mendukung VLSM/CIDR)
Pengiriman Update Broadcast 255.255.255.255 Multicast 224.0.0.9
Autentikasi Tidak ada Mendukung autentikasi plaintext dan MD5
Kompatibilitas IPv4 saja IPv4 saja (RIPng untuk IPv6)

Keunggulan

  • Sederhana dan mudah dikonfigurasi.
  • Kompatibel dengan hampir semua perangkat.

Kekurangan

  • Skalabilitas rendah (maksimal 15 hop).
  • Konvergensi lambat.
  • Update berkala menambah overhead jaringan.

CONTOH KASUS

  • Tujuan: Mengimplementasikan RIPv2 pada jaringan MikroTik untuk mendistribusikan routing dinamis antar-router secara otomatis.
  • Topologi:
    • Terdapat 5 router (RB1â€“RB5) yang saling terhubung dengan jaringan point-to-point (/30).
    • Setiap router memiliki jaringan LAN yang berbeda, yang akan didistribusikan menggunakan RIPv2.
    • Router RB1 sebagai pintu gerbang utama menuju jaringan 192.168.190.0/24.
  • Catatan:
    • Menggunakan RIPv2 dengan mode receive=v2 di setiap interface.
    • Setiap router mengiklankan network masing-masing agar dapat saling terhubung.
    • Dengan RIPv2, setiap perubahan routing akan otomatis terdistribusi ke seluruh router.

RB1


/ip address
add address=192.168.190.110/24 interface=ether1 network=192.168.190.0   # IP ke LAN/ISP
add address=192.168.1.1/30 interface=ether3 network=192.168.1.0         # Link ke RB2
add address=192.168.10.1/30 interface=ether2 network=192.168.10.0       # Link ke RB2

/ip dhcp-server network
add address=192.168.1.0/30 gateway=192.168.1.1                          # DHCP scope untuk segmen ether3

/routing rip interface
add interface=ether2 receive=v2                                         # Aktifkan RIPv2 di ether2

/routing rip network
add network=192.168.1.0/30                                              # Masukkan network ke RIP
add network=192.168.10.0/30                                             # Masukkan network ke RIP

/system identity
set name=RIPv2-R1                                                       # Nama router

Ringkasan

RIP dan RIPv2 adalah protokol routing yang mudah digunakan, namun memiliki keterbatasan untuk jaringan besar. RIPv2 memperbaiki kelemahan RIP v1 dengan mendukung subnet mask, VLSM, dan autentikasi.

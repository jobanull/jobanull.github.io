Internet Protocol Version 6 (IPv6)

IPv6 adalah generasi terbaru dari Internet Protocol yang dirancang untuk menggantikan IPv4. Tujuan utamanya adalah menyediakan ruang alamat yang sangat besar, menyederhanakan pemrosesan paket, dan meningkatkan otomatisasi konfigurasi jaringan.

Mengapa IPv6?

  • Ruang alamat masif (128-bit): ~3.4×1038 alamat.
  • Otomatisasi: SLAAC (stateless) & DHCPv6 (stateful).
  • Header lebih sederhana & dukungan extension headers.
  • Keamanan & operasi: ICMPv6, ND, dan dukungan IPsec (wajib diimplementasi).

Format Alamat & Penulisan

  • Hexadecimal 128-bit, dipisah “:”, contoh 2001:db8:85a3:0000:0000:8a2e:0370:7334.
  • Kompresi nol: hapus nol depan tiap blok (0370 → 370), ganti rangkaian nol dengan :: sekali saja, contoh 2001:db8:85a3::8a2e:370:7334.
  • Prefix: notasi CIDR, mis. 2001:db8::/32.

Tipe & Ruang Alamat

  • Global Unicast (publik): umumnya 2000::/3.
  • Link-Local: fe80::/10 (otomatis, tidak di-route, wajib ada di setiap interface).
  • Unique Local Address (ULA): fc00::/7 (praktisnya fdxx:), mirip RFC1918.
  • Multicast: ff00::/8 (mengganti broadcast di IPv4).
  • Anycast: satu alamat dipakai banyak node, dikirim ke node “terdekat”.

Prinsip Subnetting IPv6

  • /64 untuk segmen host (SLAAC butuh /64).
  • Penyedia sering memberi /48 atau /56 untuk site; Anda pecah ke banyak /64.
  • EUI-64 vs Privacy Extensions (RFC 4941) untuk membentuk Interface ID.

ICMPv6 & Neighbor Discovery (ND)

  • ND (ganti ARP): Neighbor Solicitation/Advertisement, Router Solicitation/Advertisement.
  • DAD (Duplicate Address Detection) memastikan alamat tidak duplikat.
  • Penting: jangan blokir ICMPv6 sembarangan; banyak fitur inti IPv6 bergantung padanya.

Otomatisasi Alamat

  • SLAAC (Stateless): host membentuk alamat dari RA (Router Advertisement) + /64.
  • DHCPv6 (Stateful/Stateless): bagi yang butuh kontrol lease/opsi (DNS, domain, dsb.).
  • Prefix Delegation: CPE mendapatkan prefix (mis. /56) dari ISP, lalu menyebar ke LAN.

Perbedaan Utama IPv4 vs IPv6

  • Tidak ada broadcast; gunakan multicast.
  • Router tidak mem-fragment paket; host melakukan Path MTU Discovery (MTU min 1280).
  • Header lebih ringkas; opsi dipindah ke extension headers.

Transisi & Konektivitas

  • Dual-Stack: IPv4 dan IPv6 berjalan berdampingan (metode paling umum).
  • Tunneling: GRE/IP-in-IP over IPv6/IPv4, 6rd (provider), 6to4 (legacy).
  • NAT64/DNS64: akses IPv4-only dari klien IPv6-only.

Routing IPv6 (Sekilas)

  • OSPFv3, EIGRP for IPv6, IS-IS multi-topology.
  • BGP multiprotocol (AFI/SAFI) untuk IPv6 global routing.

Keamanan & Praktik Baik

  • Aktifkan RA Guard & DHCPv6 Guard di switch untuk cegah iklan palsu.
  • Filter inbound dengan prefix-list/ACL IPv6; izinkan ICMPv6 esensial.
  • Gunakan ULA + overlay atau segmentation bila butuh isolasi internal.

CONTOH KASUS

  • Tujuan: Menghubungkan RB1, RB2, dan RB3 menggunakan protokol routing IS-IS untuk pertukaran routing.
  • RB1: Menetapkan IP untuk semua interface, membuat instance IS-IS dengan System-ID 0000.0000.0001, Level-1 & Level-2, dan NAT keluar ke internet.
  • RB2: Menetapkan IP, membuat instance IS-IS System-ID 0000.0000.0002, semua interface Level-1.
  • RB3: Menetapkan IP, membuat instance IS-IS System-ID 0000.0000.0003, semua interface Level-1.
  • Catatan: Level-1 digunakan untuk internal, Level-2 untuk eksternal/default route.

RB1


/interface 6to4
add !keepalive local-address=10.10.10.1 name=6to4-tunnel1 remote-address=\
    10.10.10.2
/ip address
add address=10.10.10.1/30 interface=ether1 network=10.10.10.0
/ipv6 address
add address=2001:a:b:10::1 interface=ether2
/ipv6 route
add distance=1 dst-address=2001:a:b:20::/64 gateway=6to4-tunnel1
/system identity
set name=RB1

RB2


/interface 6to4
add !keepalive local-address=10.10.10.2 name=6to4-tunnel2 remote-address=\
    10.10.10.1
/ip address
add address=10.10.10.2/30 interface=ether1 network=10.10.10.0
/ipv6 address
add address=2001:a:b:20::1 interface=ether2
/ipv6 route
add distance=1 dst-address=2001:a:b:10::/64 gateway=6to4-tunnel2
/system identity
set name=RB2

RB3


/ipv6 address
add address=2001:a:b:10::2 interface=ether1
/ipv6 route
add distance=1 dst-address=2001:a:b:20::/64 gateway=2001:a:b:10::1
/system identity
set name=RB3

RB4


/ipv6 address
add address=2001:a:b:20::2 interface=ether1
/ipv6 route
add distance=1 dst-address=2001:a:b:10::/64 gateway=2001:a:b:20::1
/system identity
set name=RB4

Troubleshooting Cepat

  • ping6, traceroute6 / tracert -6.
  • show ipv6 interface, show ipv6 neighbors (Cisco); MikroTik: /ipv6 neighbor print.
  • Cek RA/DHCPv6, MTU, dan izin ICMPv6 (RS/RA, NS/NA) di firewall.

Ringkasan

IPv6 menyelesaikan keterbatasan IPv4 dengan ruang alamat besar, otomatisasi konfigurasi, dan arsitektur yang lebih bersih. Gunakan /64 per segmen, pertahankan ICMPv6 penting, dan terapkan dual-stack sebagai jalur adopsi yang mulus.

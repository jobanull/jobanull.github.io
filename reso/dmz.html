Demilitarized Zone (DMZ)

DMZ (Demilitarized Zone) adalah sebuah area jaringan yang ditempatkan di antara jaringan internal (LAN) dan jaringan eksternal (internet). DMZ berfungsi sebagai buffer zone untuk memisahkan layanan publik seperti web server, mail server, atau DNS server agar dapat diakses dari luar tanpa membuka langsung akses ke jaringan internal perusahaan.

Fungsi Utama DMZ

  • Isolasi Keamanan – Memisahkan server publik dari jaringan internal.
  • Kontrol Akses – Hanya port/layer tertentu yang boleh diakses dari luar.
  • Mitigasi Serangan – Jika server DMZ diserang, LAN tetap terlindungi.
  • Penyediaan Layanan Publik – Web, email, atau DNS server tetap dapat diakses dari internet.
  • Monitoring – Lalu lintas ke/dari DMZ dapat dipantau menggunakan IDS/IPS atau firewall.

Cara Kerja DMZ

Konsep DMZ biasanya menggunakan dua firewall atau satu firewall dengan tiga interface (WAN, LAN, DMZ). Lalu lintas dari internet diarahkan ke DMZ, dan komunikasi dari DMZ ke LAN sangat dibatasi untuk mencegah kompromi jaringan internal.

Kelebihan DMZ

  • Meningkatkan keamanan dengan memisahkan server publik dari jaringan internal.
  • Memberikan fleksibilitas dalam memberikan layanan publik tanpa mengorbankan keamanan LAN.
  • Dapat digunakan untuk monitoring lalu lintas berbahaya (honeypot, IDS/IPS).

Kekurangan DMZ

  • Memerlukan perangkat firewall/router dengan konfigurasi lebih kompleks.
  • Membutuhkan biaya tambahan untuk infrastruktur dan pemeliharaan.
  • Jika salah konfigurasi, justru bisa membuka celah keamanan.

CONTOH KASUS

  • Tujuan: Mengonfigurasi DMZ pada MikroTik dan Cisco agar web server publik dapat diakses dari internet tanpa membuka akses ke jaringan internal.
  • MikroTik: Membuat interface khusus DMZ, memberikan IP DMZ, menambahkan NAT/Firewall rule agar akses dari internet diarahkan ke web server DMZ.
  • Cisco: Menggunakan firewall dengan tiga zona (Inside, Outside, DMZ) untuk memisahkan jalur lalu lintas.

Mikrotik


/ip address
add address=192.168.10.1/24 interface=ether2 comment="LAN"
add address=172.16.10.1/24 interface=ether3 comment="DMZ"
add address=10.10.10.2/30 interface=ether1 comment="WAN"

/ip firewall nat
add chain=dstnat protocol=tcp dst-port=80 \
    action=dst-nat to-addresses=172.16.10.10 to-ports=80 \
    comment="Redirect HTTP traffic to Web Server in DMZ"

/ip firewall filter
add chain=forward src-address=172.16.10.0/24 dst-address=192.168.10.0/24 \
    action=drop comment="Block DMZ to LAN"

Cisco ASA


interface GigabitEthernet0/0
 nameif outside
 security-level 0
 ip address 10.10.10.1 255.255.255.252

interface GigabitEthernet0/1
 nameif inside
 security-level 100
 ip address 192.168.10.1 255.255.255.0

interface GigabitEthernet0/2
 nameif dmz
 security-level 50
 ip address 172.16.10.1 255.255.255.0

! NAT agar server DMZ dapat diakses publik
object network WEB-SERVER
 host 172.16.10.10
 nat (dmz,outside) static 203.0.113.10

! ACL: hanya izinkan HTTP/HTTPS ke DMZ
access-list OUTSIDE_IN permit tcp any host 203.0.113.10 eq 80
access-list OUTSIDE_IN permit tcp any host 203.0.113.10 eq 443
access-group OUTSIDE_IN in interface outside

Tips Praktis

  • Gunakan firewall dengan tiga zone (Inside, Outside, DMZ) untuk kontrol akses granular.
  • Hanya buka port yang benar-benar diperlukan di DMZ.
  • Selalu lakukan patching rutin pada server di DMZ karena terekspos internet.
  • Implementasikan IDS/IPS untuk memonitor lalu lintas mencurigakan.
